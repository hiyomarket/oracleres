# 神喻穿搭引擎 V3.0 升級分析報告

**撰寫者：** 主系統 Agent  
**日期：** 2026-04-02  
**依據：** Issue #2 + `outfit-engine-v3-upgrade.md`（命理顧問 Agent）  
**範圍：** `outfitStrategy.ts`、`strategyEngine.ts`、`wuxingEngine.ts`

---

## 一、命理顧問的三項核心要求

命理顧問在 Issue #2 中提出了三項修改要求，核心目標是讓穿搭引擎從「計算機器」升級為「真正懂命理的顧問」。以下逐項分析現狀問題與修改方案。

### 要求一：五行上限保護機制

命理顧問指出，當前系統在某五行已經過旺（例如火達 51%）時，仍基於「缺土補火（火生土）」的靜態邏輯推薦火色系穿搭，這在命理上會導致能量失衡加劇。

| 狀態 | 閾值範圍 | 觸發行為 |
|------|----------|----------|
| 正常 | < 35% | 允許正常補強 |
| 充足 | 35% - 40% | 停止補強，轉向補其他弱項 |
| 過旺 | 40% - 45% | 進入保護模式，禁止推薦該元素對應顏色 |
| 嚴重過旺 | > 45% | 保護模式 + 觸發剋制平衡（配件加入剋制色） |

**現狀問題：** `outfitStrategy.ts` 的 `generateOutfitAdviceV11()` 函數完全沒有接收五行比例參數，它直接從 `OUTFIT_STRATEGY_BY_TENGOD` 靜態表取顏色。例如食神日永遠推薦「朱紅/火焰橙」上衣，不管當日火能量是 5% 還是 51%。`wuxingEngine.ts` 的 `generateOutfitAdviceV9()` 雖然有 `overflowing >= 0.35` 的避開邏輯，但只是在「避開」清單中列出過旺五行，並未主動替換穿搭顏色。`strategyEngine.ts` 的 `ENEMY_DOMINANT = 0.40` 閾值只針對忌神，不處理喜用神過旺的情況。

**命理學依據：** 根據八字命理的「五行過旺用克還是用泄」原則，當五行數量在 3-5 個（旺但不至強旺）時，適合用「克」的方式駕馭；當五行數量超過 5 個（強旺）時，必須用「泄」的方式引化能量，因為硬克容易被反克（土多木折、木多金缺、水多土流、金多火熄、火多水灼）。這個原則直接對應了命理顧問提出的閾值分級策略。

### 要求二：流日脈絡分析層

命理顧問要求系統必須向用戶解釋「為什麼今天會有這樣的五行狀態」以及「這個狀態會持續多久」，核心是區分三種能量性質：

**根旺（Rooted Prosperity）** 指本命、大運、流年三層都支持該五行，這是命主穩固的長期能量。當根旺過旺時，必須採取剋制平衡策略，因為這股能量不會自然消退。

**借旺（Borrowed Prosperity）** 指本命該五行偏弱，但流月或流日帶來了短暫的強大能量。這是宇宙給予的「視窗期」，應引導用戶順勢利用、最大化輸出，而不是刻意剋制。

**虛旺（False Prosperity）** 指僅靠流日單一維度撐起的旺度，能量極短暫，最需要把握當下。

**現狀問題：** 系統目前完全沒有能量性質判定邏輯。`wuxingEngine.ts` 雖然分別計算了本命五行（`NATAL_ELEMENT_RATIO`）和環境五行，但在加權合併後就失去了「能量來自哪裡」的資訊。`generateReasoningText()` 只有 5 種策略對應的固定文案模板，無法解釋能量的來源與持續時間。

### 要求三：推理文案升級

命理顧問要求文案必須結合 `dailyContextAnalysis` 的結果，給出有理有據的解釋。文案範例展示了一種「先說現象、再說原因、最後給行動建議和時間感」的結構。

**現狀問題：** `generateReasoningText()` 的文案是策略名稱到固定句子的映射，沒有結合具體的五行百分比數據，沒有解釋能量來源，也沒有時間感提示。

## 二、獨立研究發現與額外建議

### 發現一：日主標記不一致

系統代碼中存在日主標記的不一致。`outfitStrategy.ts` 第 4 行和 `wuxingEngine.ts` 第 5 行都標記日主為「甲木」，但用戶的知識庫明確記載日主為「庚金」。十神對應也因此不同：在庚金日主體系下，火是官殺、土是印星、金是比劫；而代碼中的十神穿搭策略是基於甲木日主的（火是食傷、土是財星、金是官殺）。

本命五行比例也存在差異：`wuxingEngine.ts` 記載為木 42%、水 35%、火 11%、土 9%、金 4%，而用戶知識庫記載為水 48%、木 32%、火 7%、土 10%、金 3%。

**建議：** 由於系統已經基於甲木日主的邏輯運行了一段時間，且十神穿搭策略表（食神、傷官、偏財等）都是按甲木日主設計的，本次升級暫不修改日主標記，但在代碼註釋中標明此差異，並建議未來與命理顧問確認後統一。

### 發現二：strategyEngine 已有部分保護但不完整

`strategyEngine.ts` 的「借力打力」策略在忌神過旺（≥40%）時會觸發剋制邏輯，但這只針對忌神（水、木），不處理喜用神（火、土、金）過旺的情況。命理顧問的要求是對**所有五行**都要有上限保護，包括喜用神。

### 發現三：兩套穿搭建議系統的整合問題

系統目前有兩套穿搭建議生成路徑：`wuxingEngine.ts` 的 `generateOutfitAdviceV9()` 和 `outfitStrategy.ts` 的 `generateOutfitAdviceV11()`。前者基於五行比例動態計算，後者基於十神靜態表。本次升級應確保兩套系統都有上限保護機制，並且 V11 版本能夠接收五行比例數據。

## 三、實作方案

### 修改一：`outfitStrategy.ts` — 五行上限保護機制

在 `generateOutfitAdviceV11()` 函數中新增 `weightedElements` 參數，加入 `checkElementProtection()` 閾值掃描函數。當某五行超過閾值時，動態替換該五行對應的穿搭顏色。

```typescript
// 新增：五行上限保護閾值
const ELEMENT_SUFFICIENT = 0.35;  // 充足：停止補強
const ELEMENT_OVERPOWER = 0.40;   // 過旺：保護模式
const ELEMENT_CRITICAL = 0.45;    // 嚴重過旺：剋制平衡

// 新增：五行相剋關係（用於剋制平衡）
const ELEMENT_COUNTER: Record<string, string> = {
  火: '水', 水: '土', 土: '木', 木: '金', 金: '火',
};
```

### 修改二：`outfitStrategy.ts` — 流日脈絡分析層

新增 `DailyContextAnalysis` 介面和 `determineSourceNature()` 函數，比較本命五行、環境五行、大運影響，判定根旺/借旺/虛旺。

```typescript
export interface DailyContextAnalysis {
  dominantElement: string;
  sourceNature: '根旺' | '借旺' | '虛旺';
  keyMessage: string;
  urgency: '把握今日' | '長期方向' | '今日收斂';
}
```

判定邏輯：若本命該五行 ≥ 20% 且大運也支持 → 根旺；若本命該五行 < 15% 但環境帶旺 → 借旺；若僅流日單維度帶旺 → 虛旺。

### 修改三：`outfitStrategy.ts` — 推理文案升級

重構 `generateReasoningText()`，結合 `dailyContextAnalysis` 生成有命理深度的文案。不同能量性質使用不同的文案風格：根旺偏「收斂穩健」，借旺偏「把握機遇」，虛旺偏「珍惜當下」。

### 修改四：`wuxingEngine.ts` — 上限保護整合

在 `generateOutfitAdviceV9()` 中強化現有的 `overflowing` 邏輯，確保過旺五行不僅被列入避開清單，還會觸發主動的顏色替換。

## 四、影響範圍評估

| 檔案 | 修改程度 | 影響範圍 |
|------|----------|----------|
| `server/lib/outfitStrategy.ts` | 重大 | 新增保護機制、脈絡分析、文案升級 |
| `server/lib/wuxingEngine.ts` | 中等 | 強化上限保護、匯出本命比例供比較 |
| `server/lib/strategyEngine.ts` | 輕微 | 無需修改，但保護機制與其互補 |
| `server/routers.ts` 或相關路由 | 中等 | 需傳遞五行比例到穿搭函數 |
| 前端穿搭頁面 | 中等 | 需展示 `dailyContextAnalysis` |

---

## 五、結論

命理顧問的三項要求都是合理且必要的。五行上限保護機制解決了「過旺仍補」的根本性錯誤；流日脈絡分析層讓系統具備了「解釋為什麼」的能力；推理文案升級則讓建議從冰冷的計算結果變成有溫度的命理指導。這三項升級將使穿搭引擎從「知道穿什麼」進化為「知道為什麼今天要這樣穿」。
