## 一、命理顧問報告摘要

命理顧問在 `yangzhai-advisor-review.md` 中提出了三項核心補充，直指目前系統在陽宅開運領域的結構性缺陷。報告的核心論點是：**一個合格的風水系統必須同時處理「理氣」（八宅方位吉凶）和「巒頭」（形煞環境識別），並且在兩者發生衝突時有明確的優先級裁決機制。** 以下是三項補充的精要：

| 補充項目 | 核心問題 | 建議方案 | 對應新模組 |
|----------|----------|----------|------------|
| 八宅與五行衝突解決 | 八宅吉位的五行可能與八字喜忌矛盾 | 「方位從八宅，佈置從五行」原則 | `fengshuiConflictResolver.ts` |
| 流日方位吉凶動態加權 | 每日干支影響方位吉凶，但系統只有靜態分析 | 結合流日地支沖煞，動態調整方位建議 | `dailyDirectionEngine.ts` |
| AI 視覺解析的形煞優先級 | 照片無法判斷精確方位，且形煞識別缺乏命理邊界 | 強制陀螺儀定向，形煞優先於理氣 | `formShaDetector.ts` Prompt 優化 |

顧問特別強調的大原則是：

> 形煞 > 理氣。一旦識別到背後無靠、上方壓樑、壁刀沖射三大職場形煞，無論理氣計算結果多吉，都必須優先化煞。這是風水界的共識。

## 二、目前系統架構盤點

經過完整的程式碼審查，目前天命共振系統在風水領域已具備一定基礎，但距離「專業權威的陽宅開運功能」仍有顯著差距。以下是現有模組與缺口的對照分析：

### 2.1 已有模組

| 檔案路徑 | 功能描述 | 可複用性 |
|----------|----------|----------|
| `server/lib/fengShuiAnalysis.ts` | 24 山方位五行計算、GPS 方位角、地名字根五行、商業類型五行匹配 | **高** — 方位計算和五行匹配邏輯可直接複用 |
| `server/lib/wuxingEngine.ts` | 五行計算核心（`BRANCH_ELEMENT`、`STEM_ELEMENT`、流日天干地支、環境元素計算） | **高** — `dayBranch` 是流日沖煞的基礎 |
| `server/lib/strategyEngine.ts` | 每日策略引擎（`DailyStrategyObject`） | **中** — 可擴展加入方位吉凶欄位 |
| `server/lib/outfitStrategy.ts` | 穿搭引擎 V3.0（五行保護機制、流日脈絡分析） | **中** — 保護機制的設計模式可借鑑 |
| `client/src/components/NearbyRestaurants.tsx` | 前端風水分析 UI（餐廳方位五行排序、風水等級顯示） | **低** — UI 參考，但場景不同 |

### 2.2 關鍵缺口

目前系統存在六項結構性缺口，必須在開發陽宅功能前逐一填補：

**第一，無八宅命卦計算。** 系統完全沒有根據出生年份計算命卦（東四命/西四命）的邏輯。八宅是陽宅風水最基礎的理論框架，缺少它等於整個方位吉凶系統沒有根基。命卦決定了一個人的四吉位（生氣、天醫、延年、伏位）和四凶位（絕命、五鬼、六煞、禍害），這是所有後續分析的前提。

**第二，無流日地支沖煞方位計算。** `wuxingEngine.ts` 雖然已有 `dayBranch`（流日地支），但沒有將其轉換為「今日哪個方位被沖」的邏輯。六沖對照（子午沖、丑未沖、寅申沖、卯酉沖、辰戌沖、巳亥沖）是命理學的基本功，每個地支對應一個方位，沖則該方位當日不吉。

**第三，無形煞識別系統。** 目前沒有任何 AI 照片分析的形煞邏輯。命理顧問明確指出三大職場形煞（背後無靠、上方壓樑、壁刀沖射）必須作為最高優先級識別項目，且形煞判定必須優先於理氣計算。

**第四，`fengShuiAnalysis.ts` 只服務餐廳場景。** 現有的風水分析引擎是為 NearbyRestaurants 組件設計的，計算的是「從用戶位置到餐廳的方位五行」，而非「用戶居家/辦公室的方位佈局」。需要一個通用的陽宅風水引擎。

**第五，本命五行比例寫死。** `NATAL_ELEMENT_RATIO` 是蘇祐震先生的固定值（木 42%、水 35%、火 11%、土 9%、金 4%），無法服務其他用戶。雖然 v5.29 已在 `getOutfitV3` 中實現了從用戶 profile 動態讀取，但陽宅功能也需要同樣的動態化。

**第六，作戰室缺少方位卡片。** WarRoom 頁面目前有穿搭、飲食、策略等模塊，但沒有「今日辦公室方位吉凶」的卡片。命理顧問建議將陽宅功能做成每日高頻工具，而非一次性靜態報告。

## 三、命理學研究補充

### 3.1 八宅命卦計算方法

八宅風水的核心是根據出生年份計算**命卦**（也稱本命卦或年命卦），將人分為**東四命**（離、坎、震、巽）和**西四命**（乾、坤、艮、兌）兩大類。計算公式如下：

| 性別 | 計算公式 | 說明 |
|------|----------|------|
| 男性 | (100 - 出生年後兩位) / 9 取餘數 | 餘數對應卦象，餘 0 為坎 |
| 女性 | (出生年後兩位 - 4) / 9 取餘數 | 餘數對應卦象，餘 0 為艮 |

餘數與卦象對照：1=坎、2=坤、3=震、4=巽、5=男坤女艮、6=乾、7=兌、8=艮、9=離。

東四命（坎、離、震、巽）的吉位在東、南、東南、北；西四命（乾、坤、艮、兌）的吉位在西、西北、西南、東北。每個命卦對應的八個方位各有固定的吉凶星：生氣（貪狼木）、天醫（巨門土）、延年（武曲金）、伏位（輔弼木）為四吉，絕命（破軍金）、五鬼（廉貞火）、六煞（文曲水）、禍害（祿存土）為四凶 [1]。

### 3.2 衝突解決：「方位從八宅，佈置從五行」

當八宅吉位的五行屬性與個人八字喜忌發生矛盾時，命理界的通行做法是**不改變方位選擇，而是通過佈置物品的五行屬性來化解**。具體而言：

以蘇祐震先生為例，他是甲木日主、水大木漂的命格，最忌水和木過旺。假設他的命卦吉位在北方（水位），直接使用北方會加重水氣。此時的化解方案是：仍然使用北方吉位（八宅優先），但在該方位佈置火屬性（紅色系）或土屬性（黃色系、陶瓷）的物品，用五行通關法將水氣轉化為有利能量。通關路徑為：水 → 木 → 火，或水 → 土（直接剋制），視命格需求而定 [2]。

### 3.3 流日地支沖煞方位

十二地支各對應一個方位，六沖則意味著兩個方位的能量直接對撞。在流日層面，當日地支所沖的方位當天不宜進行重要活動或長時間停留。

| 流日地支 | 沖對地支 | 不吉方位 | 方位五行 |
|----------|----------|----------|----------|
| 子（北） | 午（南） | 南方 | 火 |
| 丑（東北） | 未（西南） | 西南 | 土 |
| 寅（東北） | 申（西南） | 西南 | 金 |
| 卯（東） | 酉（西） | 西方 | 金 |
| 辰（東南） | 戌（西北） | 西北 | 金 |
| 巳（東南） | 亥（西北） | 西北 | 水 |

此外，三煞方位（年煞、月煞、日煞）也會影響方位吉凶。三煞的計算基於地支三合局：申子辰年三煞在南（巳午未），寅午戌年三煞在北（亥子丑），亥卯未年三煞在西（申酉戌），巳酉丑年三煞在東（寅卯辰）[3]。

### 3.4 三大職場形煞識別標準

根據命理專家李行的實地解析 [4] 和命理顧問的補充，辦公室三大形煞的識別標準和化解方案如下：

**背後無靠**是最常見的職場形煞。座位後方為窗戶、門口或走道時，氣場不穩、專注力下降，容易犯小人。關鍵判斷標準不是「後方有沒有牆」，而是「是否有強烈氣流或頻繁人流從背後通過」。化解方案包括使用高背椅、在椅背貼黃色硬紙、或在背後放置屏風。

**上方壓樑**源於視覺壓迫和心理暗示。頭頂有橫樑會造成壓力感，影響晉升和業績。化解的核心是「先改善視覺，再擺風水物」——在桌面放置由下往上打光的燈具，讓壓樑處變亮，產生天花板變高的錯覺。

**壁刀沖射**指牆角、柱角直對座位方向，形成「刀煞」。這類煞氣會導致口舌是非和健康問題。化解方案是用盆栽或屏風遮擋尖角，阻斷煞氣直射路徑。

## 四、開發計畫

基於以上分析，陽宅開運功能的開發分為三個階段，每個階段都對應命理顧問報告中的一項核心要求。

### 階段一：八宅命卦引擎 + 衝突解決機制

本階段建立陽宅風水的理論基礎，新增兩個核心模組：

| 新增檔案 | 功能 | 關鍵邏輯 |
|----------|------|----------|
| `server/lib/bazhai.ts` | 八宅命卦計算引擎 | 根據出生年份和性別計算命卦（東四命/西四命），輸出八方位吉凶星配置 |
| `server/lib/fengshuiConflictResolver.ts` | 衝突解決機制 | 比對八宅吉位五行與八字喜忌，當衝突時輸出「方位不變 + 五行通關佈置方案」 |

`bazhai.ts` 的核心輸出是一個 `BazhaiResult` 物件，包含命卦名稱、東四/西四命分類、八個方位各自的吉凶星和五行屬性。`fengshuiConflictResolver.ts` 接收 `BazhaiResult` 和用戶的五行喜忌，逐一檢查每個吉位是否與喜忌衝突，若衝突則生成通關方案（具體的佈置顏色、材質、物品建議）。

### 階段二：流日方位吉凶動態引擎

本階段讓陽宅功能從靜態報告升級為每日動態工具：

| 新增/修改檔案 | 功能 | 關鍵邏輯 |
|---------------|------|----------|
| `server/lib/dailyDirectionEngine.ts` | 流日方位吉凶計算 | 根據當日地支計算六沖方位、三煞方位，與八宅吉凶疊加後輸出「今日八方位綜合評分」 |
| `server/routers.ts` 新增 `getDailyDirections` | 每日方位 API | 整合八宅 + 流日沖煞 + 五行衝突解決，返回完整的今日方位建議 |
| `client/src/pages/WarRoom.tsx` 新增方位卡片 | 今日辦公室方位吉凶 | 八方位羅盤圖 + 吉凶標籤 + 避坑提醒 + 佈置建議 |

動態加權的計算邏輯是：八宅基礎分（0-100）× 流日修正係數（0.5-1.2）。當流日地支沖到某方位時，該方位的修正係數降為 0.5（即吉度減半）；當流日地支生助某方位時，修正係數升為 1.2。三煞方位的修正係數固定為 0.3（強烈不吉）。

### 階段三：AI 視覺形煞識別

本階段加入照片分析功能，讓用戶拍攝辦公桌/居家環境照片，系統自動識別形煞：

| 新增/修改檔案 | 功能 | 關鍵邏輯 |
|---------------|------|----------|
| `server/lib/formShaDetector.ts` | 形煞識別 Prompt 工程 | 調用 GPT-4V，優先識別三大形煞，輸出形煞類型 + 嚴重度 + 化解方案 |
| `server/routers.ts` 新增 `analyzeOfficePhoto` | 照片分析 API | 接收照片 URL + 陀螺儀方位角，返回形煞報告 + 方位建議 |
| `client/src/pages/YangzhaiAnalysis.tsx` | 陽宅分析頁面 | 照片上傳 + 陀螺儀定向 + 形煞報告 + 方位羅盤 + 佈置建議 |

GPT-4V 的 Prompt 設計遵循命理顧問的「形煞優先級」原則：系統指令明確要求模型**先掃描三大形煞（背後無靠、上方壓樑、壁刀沖射），再分析其他風水元素**。一旦偵測到形煞，報告的第一段必須是形煞警告和化解方案，理氣分析降為次要。

方位判斷方面，系統**不依賴照片推斷方位**（命理顧問明確指出照片無法判斷 24 山方位），而是強制要求用戶提供手機陀螺儀/指南針的方位角數據。前端使用 `DeviceOrientationEvent` API 獲取手機朝向，結合 GPS 定位，精確計算座位面向的 24 山方位。

## 五、優先級建議與風險評估

### 5.1 開發優先級

| 優先級 | 項目 | 預估工作量 | 依賴關係 |
|--------|------|------------|----------|
| P0 | `bazhai.ts` 八宅命卦引擎 | 中 | 無 |
| P0 | `fengshuiConflictResolver.ts` 衝突解決 | 中 | 依賴 bazhai.ts |
| P1 | `dailyDirectionEngine.ts` 流日方位 | 中 | 依賴 bazhai.ts + wuxingEngine |
| P1 | WarRoom 方位卡片 | 中 | 依賴 dailyDirectionEngine |
| P2 | `formShaDetector.ts` AI 形煞識別 | 高 | 依賴 LLM API + 前端陀螺儀 |
| P2 | `YangzhaiAnalysis.tsx` 完整分析頁 | 高 | 依賴所有後端模組 |

### 5.2 風險評估

**陀螺儀相容性風險。** `DeviceOrientationEvent` 在 iOS 13+ 需要用戶授權，部分 Android 裝置的指南針精度不足。建議提供手動輸入方位角的備選方案。

**八宅流派差異風險。** 八宅風水有多種流派（翻卦法、大遊年法等），計算結果可能略有差異。建議採用最廣泛認可的大遊年法，並在報告中註明所用流派。

**AI 形煞識別準確度風險。** GPT-4V 對建築結構的識別可能有誤判（例如將裝飾性橫樑誤判為結構性壓樑）。建議在報告中加入「AI 分析僅供參考，建議搭配專業風水師實地勘察」的免責聲明。

## 六、與目前系統的整合點

陽宅開運功能不是獨立的孤島，而是與現有系統深度整合的：

**與穿搭引擎 V3.0 的聯動。** 穿搭引擎的五行保護機制（>45% 過旺觸發保護）可以直接應用於陽宅場景。當流日某五行過旺時，不僅穿搭要避開該五行顏色，辦公室佈置也要避開該五行的方位和物品。兩個系統共享同一套 `checkElementProtection` 邏輯。

**與作戰室的整合。** WarRoom 已有穿搭、飲食、策略模塊，新增「今日方位吉凶」卡片是自然的擴展。卡片顯示一個簡化的八方位羅盤，用紅/黃/綠標記今日各方位的吉凶程度，點擊可展開詳細的佈置建議。

**與專家市場的連結。** 陽宅分析報告的底部可以推薦「風水」類別的專家，讓用戶在 AI 分析後可以預約專業風水師進行深度諮詢。這也為專家市場帶來新的流量入口。
