> 報告日期：2026-03-27
> 版本：M4（六大系統平衡）+ M5（五行屬性升級成長系統）

## 一、問題診斷

根據墟界魔物數值平衡分析報告和資料庫實際數據，系統存在以下核心問題：

### 1.1 怪物數值失衡

| 問題 | 具體表現 | 嚴重度 |
|------|---------|--------|
| HP 跳躍過大 | common Lv4 HP=814（應為 140）、legendary Lv50 HP=50000 | P0 |
| ATK/DEF 無差異化 | 物理型和魔法型怪物的 ATK 相同 | P1 |
| SPD 分布不合理 | 所有怪物 SPD 在 8-30 之間無規律 | P1 |
| 種族加乘缺失 | 龍族和昆蟲族 HP 相同 | P2 |

### 1.2 人物技能全部相同

| 欄位 | 現狀 | 問題 |
|------|------|------|
| powerPercent | 全部 100% | 初階和天命技能威力相同 |
| cooldown | 全部 0 | 無冷卻設計 |
| mpCost | 4~12 | 範圍太窄 |

### 1.3 裝備數值倒掛

| 稀有度 | 平均 ATK | 問題 |
|--------|---------|------|
| common | 5 | 正常 |
| rare | 10 | 正常 |
| epic | 15 | 正常 |
| legendary | 3 | 比 common 還低 |

### 1.4 商店價格嚴重失衡

| 道具 | 現有價格 | 問題 |
|------|---------|------|
| 史詩技能書 | 800 金幣 | 遠低於合理價格 |
| 史詩技能書（靈相商店） | 80 靈石 | 遠低於合理價格 |
| 傳說裝備 | 1200 金幣 | 與普通裝備差距不大 |

### 1.5 五行屬性與戰鬥數值脫節

升級時不增加五行屬性，五行屬性對戰鬥數值的影響係數過低（wood * 3 對比 level * 10），導致五行屬性形同虛設。

```
baseAttack = baseHp × typeMultiplier
  物理型: ×0.16    魔法型: ×0.12    坦克型: ×0.10    脆皮型: ×0.18

  物理型: ×0.12    魔法型: ×0.10    坦克型: ×0.20    脆皮型: ×0.06
```

## 三、五行屬性升級成長系統（M5）

### 3.1 五行對應戰鬥屬性

| 五行 | 中文名 | 主要影響 | 次要影響 |
|------|--------|---------|---------|
| 木 | 體力 | HP（×5） | DEF（×0.3） |
| 火 | 力量 | ATK（×1.2） | — |
| 土 | 強度 | DEF（×1.0） | HP（×2） |
| 金 | 速度 | SPD（×0.5） | ATK（×0.3） |
| 水 | 魔法 | MP（×3）、MATK（×1.0） | — |

### 3.2 升級時五行屬性成長

每次升級自動獲得 **5 點**五行屬性，分配規則：

| 分配對象 | 點數 | 說明 |
|---------|------|------|
| 主屬性（dominantElement） | +2.0 | 角色五行傾向決定 |
| 相生屬性 | +1.5 | 木生火、火生土、土生金、金生水、水生木 |
| 其他三屬性 | 各 +0.5 | 均勻分配 |

### 3.3 戰鬥數值推導公式（calcCharacterStatsV2）

```
HP  = 50 + wood × 5 + earth × 2 + level × 3
MP  = 30 + water × 3 + level × 2
ATK = 5 + fire × 1.2 + metal × 0.3 + level × 0.5
DEF = 3 + earth × 1.0 + wood × 0.3 + level × 0.3
SPD = 5 + metal × 0.5 + level × 0.1
MATK = 3 + water × 1.0 + fire × 0.2 + level × 0.3
```

## 四、AI 審核機制

所有 AI 生成工具（怪物、技能、裝備、道具）在生成後自動執行數值審核：

### 4.1 審核函數

| 函數名 | 審核對象 | 檢查項目 |
|--------|---------|---------|
| `auditMonsterStats` | 怪物 | HP/ATK/DEF/SPD 是否在基準值 ±35% 內 |
| `auditSkillStats` | 技能 | 威力/MP/冷卻是否在階級範圍內 |
| `auditEquipStats` | 裝備 | ATK/DEF/HP/SPD 加成是否合理 |
| `auditPrice` | 價格 | 金幣/靈石是否在基準值 ±50% 內 |

### 4.2 審核流程

```
AI 生成數值 → auditXxx() 檢查 → 
  ├─ 無違規 → 直接寫入資料庫
  ├─ warning → 自動修正到合理範圍，記錄日誌
  └─ error → 自動修正到基準值，記錄日誌
```

### 4.3 商店刷新價格審核

`tickEngine.ts` 中的 `calcPrice` 函數已更新，使用 `PRICE_TABLE` 基準表：
- 根據道具類型（技能書/裝備/消耗品/材料）和稀有度查表
- 加入 ±20% 隨機浮動
- 隱藏商店加價 50%
- 靈相商店使用靈石定價
