## 一、功能描述

在「命格設定」頁面（`/my-profile`）加入性別選擇欄位，讓用戶可以設定自己的性別（男/女/不指定）。設定後，系統在所有顯示塔羅牌圖片的頁面，自動切換為對應性別版本的插畫。

**用戶體驗改變**：
- 目前：所有用戶看到同一套塔羅牌圖片（預設為女生版）
- 改後：男性用戶看到男生版 22 張插畫，女性用戶看到女生版 22 張插畫

---

## 二、用戶操作流程

```
用戶進入 /my-profile（命格設定頁）
  ↓
在「個人資料」區塊看到「性別」欄位
（選項：男生 / 女生 / 不指定）
  ↓
選擇後點擊「儲存設定」
  ↓
系統呼叫 trpc.userProfile.updateGender.useMutation()
  ↓
儲存成功 → toast 提示「設定已儲存」
  ↓
下次進入作戰室、問卜等頁面時，塔羅牌圖片自動切換
```


**結論：資料庫層面零改動。**

---

### 3.2 後端：新增 tRPC Procedures

需要在 `server/routers.ts` 的 `userProfile` router（或 `auth` router）中新增以下兩個 procedures：

**Procedure 1：讀取用戶性別設定**

```ts
// 建議放在 auth.me 的回傳資料中（直接擴充現有 query）
// 或在 userProfile router 新增獨立 query

getGender: protectedProcedure.query(async ({ ctx }) => {
  const db = await getDb();
  if (!db) return { gender: null };
  const [profile] = await db
    .select({ gender: userProfiles.gender })
    .from(userProfiles)
    .where(eq(userProfiles.userId, ctx.user.id))
    .limit(1);
  return { gender: profile?.gender ?? null };
}),
```

**Procedure 2：更新用戶性別設定**

```ts
updateGender: protectedProcedure
  .input(z.object({
    gender: z.enum(["male", "female", "other"]),
  }))
  .mutation(async ({ input, ctx }) => {
    const db = await getDb();
    if (!db) throw new TRPCError({ code: 'INTERNAL_SERVER_ERROR' });
    await db
      .insert(userProfiles)
      .values({ userId: ctx.user.id, gender: input.gender })
      .onDuplicateKeyUpdate({ set: { gender: input.gender } });
    return { success: true };
  }),
```

**建議放置位置**：`server/routers.ts` 中的 `userProfile: router({...})` 區塊（目前只有 `yearlyAnalysis`，直接在其後新增）。

---

### 3.3 前端：塔羅牌圖片切換邏輯

**影響的核心邏輯位置**：`calculateTarotDailyCard` 函式回傳的塔羅牌物件，目前包含圖片 URL。需要確認圖片 URL 的來源位置。

根據 `routers.snapshot.ts` 的呼叫方式：
```ts
const tarot = calculateTarotDailyCard(month, day, ep.birthMonth, ep.birthDay);
// tarot.card.name, tarot.card.keywords, tarot.card.advice...
```

**推測的切換方案**（需主系統確認圖片 URL 的實際儲存位置）：

**方案 A：在 `getUserProfileForEngine` 回傳 gender**
- 在 `server/db.ts` 的 `getUserProfileForEngine` 函式中，加入 `gender` 欄位的查詢與回傳
- 在所有呼叫 `calculateTarotDailyCard` 的地方，根據 `ep.gender` 決定使用哪套圖片 URL

**方案 B：在前端根據 `auth.me` 的 gender 欄位切換**
- 在 `auth.me` 的回傳資料中加入 `gender`（從 `userProfiles` join 查詢）
- 前端元件讀取 `user.gender`，動態組合塔羅牌圖片 URL

**建議採用方案 B**（前端切換），理由：
- 塔羅牌圖片 URL 是靜態資源（CDN），在前端切換更直接
- 不需要修改後端的計算引擎
- 前端已有 `useAuth()` hook 可直接讀取 user 資料
