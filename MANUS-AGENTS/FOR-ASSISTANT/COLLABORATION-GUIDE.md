# 天命輔助系統 — 協作指南與提案規範

> 這份文件說明如何撰寫合格的功能提案，讓天命主系統能直接審核並整合。
> 最後更新：2026/03/22 by 天命主系統

---

## 提案的核心原則

你提交的提案，目標是讓天命主系統**不需要猜測、不需要大幅修改**，就能直接按照提案實作。提案越精確，整合越順暢，消耗的資源越少。

---

## 提案合格標準（必須全部符合）

天命主系統在審核提案時，會逐一檢查以下項目。任何一項不符合，提案會被退回：

| 檢查項目 | 說明 |
|----------|------|
| **使用規定格式** | 必須使用 `PROPOSALS/PROPOSAL-TEMPLATE.md` 的格式 |
| **影響範圍清楚** | 明確列出「會修改哪些現有檔案」（精確到檔案路徑） |
| **新增檔案清楚** | 明確列出「需要新建哪些檔案」（精確到檔案路徑） |
| **資料庫說明清楚** | 說明是否需要新增資料表或欄位；如有，提供 Drizzle schema 草稿 |
| **技術棧合規** | 只使用現有技術棧（React 19 / Tailwind 4 / tRPC / Drizzle） |
| **設計語言合規** | 使用現有 CSS 類別（`.glass-card`、`.text-gold-gradient` 等），不自創新設計系統 |
| **無衝突** | 不與現有功能衝突；如有重疊，說明如何共存 |

---

## 技術棧限制（不可引入的東西）

以下是**絕對不能**在提案中引入的技術，否則提案直接退回：

- 任何新的 CSS 框架（Bootstrap、Bulma 等）
- React Router（系統用 Wouter）
- Axios 或 fetch wrapper（系統用 tRPC）
- 任何 ORM（系統用 Drizzle）
- 任何新的 UI 元件庫（系統用 shadcn/ui）
- 任何需要新的環境變數的外部服務（除非 Boss 明確批准）

---

## 設計語言限制

提案中涉及 UI 的部分，必須遵守以下規則：

**必須使用的現有 CSS 類別**：
- `.glass-card` — 所有卡片元件
- `.text-gold-gradient` — 金色標題文字
- `.text-holographic` — 特殊強調文字
- `.gold-pulse` — 重要按鈕的發光效果
- `.oracle-text-gradient` — 命理相關文字

**色彩規則**：
- 背景色：`bg-background`（深藍黑）
- 主色：金色系（`text-amber-400`、`border-amber-500/30`）
- 卡片：`glass-card` 類別
- 不能使用 HEX 或 RGB 硬編碼顏色

**字體規則**：
- 標題：Noto Serif TC（繁體中文襯線）
- 內文：系統預設（Noto Sans TC）

## 提案命名規則

```
PROPOSAL-[YYYYMMDD]-[功能名稱]-PROPOSAL.md

範例：
PROPOSAL-20260322-每日簽到功能-PROPOSAL.md
PROPOSAL-20260401-命盤分析頁面-PROPOSAL.md
```

## 提案狀態說明

| 狀態 | 意義 | 由誰設定 |
|------|------|----------|
| `draft` | 草稿，尚未完成 | 輔助系統 |
| `pending` | 已完成，等待審核 | 輔助系統 |
| `approved` | 審核通過，等待整合 | 天命主系統 |
| `rejected` | 退回，需修改 | 天命主系統 |
| `integrated` | 已整合進網站 | 天命主系統 |

---

## 常見退回原因（避免這些錯誤）

1. **影響範圍不清楚**：只說「修改首頁」，沒有說具體是哪個檔案的哪個部分
2. **沒有資料庫說明**：功能需要儲存資料，但提案沒有說明資料庫異動
3. **引入新框架**：提案中出現 `npm install xxx` 的建議
4. **自創設計系統**：提案中定義了新的 CSS 變數或顏色
5. **格式不完整**：PROPOSAL-TEMPLATE.md 中的必填欄位沒有填寫
6. **與現有功能衝突**：提案的功能與現有功能重疊，但沒有說明如何處理
