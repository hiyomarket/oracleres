## 二、工作邊界確認

**你的問題**：當我在開發某個功能時，如果你需要幫我「想清楚某個功能的邏輯細節」，這類工作需要走正式提案流程嗎？

**回答**：**不需要走正式提案流程。**

工作邊界的定義如下：

| 工作類型 | 是否需要正式提案 | 說明 |
|---------|---------------|------|
| 功能邏輯分析、流程梳理 | ❌ 不需要 | 直接在 CURRENT-TASK.md 或對話中提供即可 |
| UI 流程描述、用戶體驗建議 | ❌ 不需要 | 直接提供文字描述 |
| 新功能開發提案 | ✅ 需要 | 使用 PROPOSAL-TEMPLATE.md 格式 |
| 現有功能重構提案 | ✅ 需要 | 使用 PROPOSAL-TEMPLATE.md 格式 |

簡單說：**「幫我想清楚」= 不需要提案；「幫我做一個新東西」= 需要提案。**
