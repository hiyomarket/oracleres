## 第一步：建立你對系統的基本認識

請依序讀完以下文件，不要跳過任何一份：

**第一份**：`MANUS-AGENTS/FOR-ASSISTANT/README.md`
你的角色定義、能做什麼、不能做什麼。這是你的行動邊界，必須完全理解。

**第二份**：`MANUS-AGENTS/FOR-ASSISTANT/COLLABORATION-GUIDE.md`
提案的格式規範和合格標準。未來你提交的每一份提案，都必須符合這裡的要求。

**第三份**：`SYSTEM-BACKUP/LAST-UPDATED.md`
確認目前系統的版本號，確保你讀到的是最新的備份。

**第四份**：`SYSTEM-BACKUP/routers.snapshot.ts`
這是整個系統的 API 骨架。讀完後你會知道系統目前有哪些功能、資料怎麼流動。重點關注：有哪些 procedure、輸入輸出是什麼。

**第五份**：`SYSTEM-BACKUP/schema.snapshot.ts`
資料庫的結構。讀完後你會知道系統存了哪些資料、資料表之間的關係。

**第六份**：`SYSTEM-BACKUP/App.snapshot.tsx`
前端的路由結構。讀完後你會知道系統有哪些頁面、用戶的導覽路徑是什麼。

**第七份**：`CLAW-TEAM/FEATURE-LIST.md`
完整的功能清單，包含已完成的功能和規劃中的功能。這裡有你未來可以協助發展的方向。

**第八份**：`CLAW-TEAM/ARCHITECTURE/`（資料夾內所有文件）
技術架構決策記錄。讀完後你會知道系統在重要架構選擇上的歷史決策，避免在提案中重複已被否決的方向。

**第九份**：`CLAW-TEAM/DECISIONS/`（資料夾內所有文件）
重要決策記錄。讀完後你會了解 Boss 和主系統在關鍵功能上的設計取捨，提案時能更精準地對齊方向。

## 第二步：了解這個系統在做什麼

天命共振是一個以八字命理為核心的個人化命理系統。主要用戶是一位甲木日主（1990/09/05 生），系統為他提供每日運勢、擲筊問卦、天命問卜等功能。

這個系統的特別之處在於：**所有功能都以這位用戶的命格為基礎計算**，不是通用的命理工具，而是高度個人化的命理助手。

讀完 `SYSTEM-BACKUP/userProfile.snapshot.ts`，你會看到這位用戶的完整命格資料。這是整個系統的核心，所有功能都圍繞這份資料運作。
