# 一、系統概覽

## 1.1 設計背景

本系統基於日本 Square Enix 經典 MMORPG《魔力寶貝》（Cross Gate，2001年）的戰鬥公式進行移植和改編，將原本的「地、水、火、風」四元素系統替換為「木、火、土、金、水」五行系統，並根據天命共振的遊戲設定進行適當調整。

## 1.2 核心設計原則

| 原則 | 說明 |
|-----|------|
| **數值繼承** | 核心公式係數直接移植，確保數值平衡性經過驗證 |
| **五行替換** | 四元素 → 五行，保持相剋邏輯一致性 |
| **簡化適配** | 種族數量減少，適合輕度玩家上手 |
| **延展預留** | 保留完整系統以便後續擴展 |

## 1.3 與魔力寶貝對照表

| 項目 | 魔力寶貝 | 天命共振 |
|-----|---------|---------|
| 屬性數量 | 4種（地水火風） | 5種（木火土金水） |
| 修正係數 | 0.3 | 0.3 |
| 每點剋制 | ±3.75% | ±3.75% |
| 種族數量 | 10種 | 6種 |

# 二、五行屬性系統

## 2.1 五行基礎

### 2.1.1 五行定義

五行是中國古代哲學的核心概念，代表自然界五種基本元素。所有戰鬥單位（角色、靈寵、怪物）都擁有五行屬性配置。

### 2.1.2 五行分類

| 五行 | 代表意義 | 對應魔獸元素 | 戰鬥定位 |
|-----|---------|-------------|---------|
| 木 | 生長、生命力 | 地 | 治療、輔助 |
| 火 | 熱情、破壞 | 火 | 輸出、攻擊 |
| 土 | 穩定、防禦 | 風 | 坦克、保護 |
| 金 | 堅硬、銳利 | 水 | 切割、爆發 |
| 水 | 靈活、流動 | （特殊）| 控場、干擾 |

### 2.1.3 對應替換邏輯

```
魔力寶貝：地 → 木
魔力寶貝：水 → 金
魔力寶貝：火 → 火（保留）
魔力寶貝：風 → 土
（水在五行中地位特殊，故採用直接映射）
```

## 2.2 相生相剋關係

### 2.2.1 相剋循環（戰鬥使用）

```
木 → 土 → 金 → 水 → 木
  ↑                   ↓
  └────── 相生 ────────┘
```

**說明**：
- 「→」表示前者剋制後者
- 木剋土、土剋金、金剋水、水剋木
- 火與其他四行的關係需要特別處理（見下文）

### 2.2.2 特殊關係：火

火在五行中具有特殊地位，不在標準相剋循環中：

| 五行 | 與火關係 |
|-----|---------|
| 木 | 木生火（相生）→ 火攻擊木無加成 |
| 土 | 土生金、金生水（無直接關聯）|
| 金 | 金生水（無直接關聯）|
| 水 | 水克火（相剋）→ 木屬攻擊火有加成 |

**簡化處理**：
- 火與木：無剋制關係（相生）
- 火與土：無剋制關係
- 火與金：無剋制關係
- 火與水：火剋水（+30%）

### 2.2.3 完整相剋表

| 攻擊方 | 木 | 火 | 土 | 金 | 水 |
|-------|----|----|----|----|----|
| **木** | 0% | 0% | +30% | 0% | 0% |
| **火** | 0% | 0% | 0% | 0% | +30% |
| **土** | 0% | 0% | 0% | +30% | 0% |
| **金** | 0% | 0% | 0% | 0% | 0% |
| **水** | 0% | -30% | 0% | 0% | 0% |

*註：以上為純五行（10點）攻擊純五行（10點）的修正值*

## 2.3 屬性點數配置

### 2.3.1 配置規則

- **玩家/靈寵**：總和為 10 點
- **怪物**：總和通常為 100

### 2.3.2 有效配置範例

| 配置類型 | 範例 | 說明 |
|---------|------|------|
| 純五行 | 10木 | 單一屬性極致 |
| 相生配置 | 5木5火 | 木生火，互相增益 |
| 相剋配置 | 5木5土 | 混合效果，較為平衡 |
| 無關配置 | 5木5金 | 金生水，與木無直接關聯，防禦屬性 |
| 多餘配置 | 6木6火 | 合計>10，大於10部分單獨計算 |

# 三、屬性相剋計算

## 3.1 核心修正係數

```
┌────────────────────────────────────────────────────────────┐
│                                                            │
│   五行修正係數：0.3                                         │
│                                                            │
│   公式：修正值 = 0.3 × (自身屬性點數 × 對方被剋屬性點數)   │
│                                                            │
│   範例：                                                    │
│   - 10木 攻擊 10土 → 修正 +0.30 → 傷害 +30%               │
│   - 5木 攻擊 5土 → 修正 +0.075 → 傷害 +7.5%               │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

## 3.2 計算公式

### 3.2.1 基本公式

```
修正值 = 0.3 × Σ(自身五行i × 對方五行j) - 0.3 × Σ(自身五行k × 對方五行l)

其中：
- i→j 為相剋關係（木剋土、土剋金、金剋水、水剋火、火剋木）
- k→l 為被剋關係
- 每種屬性只能參與一次計算（不重複計算）
```

### 3.2.2 詳細計算規則

1. **前後位置固定**：攻擊方為1號，防禦方為2號，不能調換
2. **單次計算**：每種屬性只能參與一次計算，不能重複
3. **符號判定**：
   - 攻擊方屬性剋制防御方屬性 → 正值（+）
   - 攻擊方屬性被防御方屬性剋制 → 負值（-）
4. **最終處理**：所有修正值相加，正值增加傷害，負值減少傷害

### 3.2.3 公式簡化版

```
當攻擊方五行點數合計 ≤ 10 且防御方五行點數合計 = 10 時：

修正值 = 0.3 × [ Σ(攻擊方屬性 × 對應剋制系數) - Σ(攻擊方屬性 × 對應被剋系數) ]

對應關係表（木→土→金→水→木）：
┌──────┬────┬────┬────┬────┬────┐
│ 攻擊方 │ 木 │ 火 │ 土 │ 金 │ 水 │
├──────┼────┼────┼────┼────┼────┤
│  木   │  0 │  0 │ +1 │  0 │ -1 │
│  火   │ -1 │  0 │  0 │  0 │ +1 │
│  土   │  0 │ -1 │  0 │ +1 │  0 │
│  金   │  0 │  0 │  0 │  0 │  0 │
│  水   │ +1 │ -1 │  0 │  0 │  0 │
└──────┴────┴────┴────┴────┴────┘

說明：+1 表示該屬性可剋制對方，-1 表示被對方剋制
```

## 3.3 實測數據對照（引用魔力寶貝實測）

### 3.3.1 魔力寶貝實驗數據

根據哈瓦游實測（2025-02-12）：

| 測試項目 | 測試方法 | 結果 |
|---------|---------|------|
| 魔法對屬性每點克制 | 用風魔法打6地4水目標 | 每點 +2% |
| 屬性每點相剋 | 裝備純風水晶打6地4水目標 | 每點 ±3.75% |
| 屬性被剋 | 裝備純水水晶打6地4水目標 | 會減少傷害 |
| 魔法無被剋機制 | 冰魔法打地屬性目標 | 無減傷，只有增益 |

### 3.3.2 數據總結

| 項目 | 數值 |
|-----|------|
| **五行相剋**（每點） | 傷害 ±3.75% |
| **法術五行克制**（每點） | 傷害 +2%（僅增益，無減傷）|
| **物理五行修正** | 0.3（全剋30%）|
| **法術五行修正** | 近似0.3 |

## 3.4 計算範例（完整推导）

### 3.4.1 範例一：6木4火 VS 5土5金

**情境**：1號（6木4火）攻擊2號（5土5金）

**Step 1：分析相剋關係**
- 1號的木（6）→ 2號的土（5）→ 木剋土 → 正向修正
- 1號的火（4）→ 2號的金（5）→ 無直接關聯 → 無修正
- 1號的火（4）→ 2號的土（5）→ 無直接關聯 → 無修正
- 1號的木（6）→ 2號的金（5）→ 無直接關聯 → 無修正

**Step 2：應用公式**
```
修正值 = 0.3 × (6/10 × 5/10)  // 木剋土
       = 0.3 × 0.3
       = 0.09
```

**Step 3：轉換為傷害修正**
```
修正 0.09 = +9% 傷害
```

**驗證**（對照魔力寶貝實測）：
- 實際剋制點數：6 × 0.5 = 3 點
- 每點 3.75% → 3 × 3.75% = 11.25%
- 接近計算結果（差異來自四捨五入）

### 3.4.2 範例二：10木 VS 10火

**情境**：1號（10木）攻擊2號（10火）

**Step 1：分析相剋關係**
- 木 → 土 → 金 → 水 → 木
- 火不在標準循環中，與木為相生關係（非相剋）

**Step 2：結論**
```
修正值 = 0（木與火無相剋關係）
傷害變化 = 0%
```

### 3.4.3 範例三：8木2火 VS 6土4水

**情境**：1號（8木2火）攻擊2號（6土4水）

**Step 1：分析**
- 1號木（8）對 2號土（6）+水（4）
  - 土被木剋：0.3×0.8×0.6 = +0.144
  - 水與木無直接關聯
- 1號火（2）對 2號土（6）+水（4）
  - 火與土無直接關聯
  - 火與水無直接關聯

**Step 2：計算**
```
修正值 = 0.144
傷害修正 = +14.4%
```

**驗證**：
- 剋制點數：8×0.6 = 4.8 點
- 4.8 × 3.75% = 18%
- 差異來自計算精度，實際可採用 15%（半剋）近似值

### 3.4.4 範例四：10木10土 VS 5土5金（大屬性）

**情境**：1號（10木10土，合計20）攻擊2號（5土5金，合計10）

**Step 1：減法處理**
- 1號合計20，2號合計10
- 先抵消：取1號的10點與2號的10點計算
- 剩餘：1號剩 10木

**Step 2：第一部分計算（抵消部分）**
```
1號：10木10土（視為5木5土）
2號：5土5金

木剋土：0.3×0.5×0.5 = +0.075
土被金剋：0.3×0.5×0.5 = -0.075

第一部分修正 = 0
```

**Step 3：第二部分計算（剩餘部分）**
```
1號剩餘：10木
與2號全部屬性計算：
- 木與土：木剋土 → 0.3×1.0×0.5 = +0.15
- 木與金：無關聯 → 0

第二部分修正 = +0.15
```

**Step 4：總修正**
```
總修正 = 0 + 0.15 = +0.15 → 傷害 +15%
```

### 3.4.5 範例五：5木5水 VS 5木5水（無關配置）

**情境**：1號（5木5水）攻擊2號（5木5水）

**Step 1：分析**
- 木與木：無關聯
- 木與水：無直接關聯
- 水與木：無直接關聯
- 水與水：無關聯

**Step 2：結論**
```
修正值 = 0
這是典型的防禦屬性配置，互相抵消
```

## 3.5 特殊屬性計算規則

### 3.5.1 無關屬性

**定義**：既不相生也不相剋的屬性組合

**範例**：
- 木+水：水生木（相生），但木與水無直接剋制
- 火+金：無直接關聯
- 土+水：無直接關聯

**計算結論**：
```
無關屬性配置 = 防禦屬性
攻擊方無法獲得明顯優勢，但也不會劣勢
```

### 3.5.2 全五行（合計屬性>10）

**定義**：四種或五種屬性同時存在

**範例**：3木3火3土3水（合計12）

**計算方法**：
1. 先抵消對方的等量屬性（最多抵消10點）
2. 剩餘屬性單獨計算修正

**範例計算**：3木3火3土3水 VS 5土5金
```
Step 1：抵消
- 3木抵消對方0（木與土金無直接關聯）
- 3火抵消對方0（火與土金無直接關聯）
- 3土抵消對方3（土被金剋，但金只有5）
- 3水抵消對方0（水與土金無直接關聯）

Step 2：剩餘
- 剩餘：3土（3-3=0），但金剩餘 5-3=2
- 實際：1號剩 0，2號剩 2金

Step 3：最終修正
- 1號無剩餘屬性 → 修正 0%
- 結論：全五行屬性被相生雙五屬性完全抵消
```

### 3.5.3 結論總結

| 屬性類型 | 特性 | 戰鬥價值 |
|---------|------|---------|
| 純五行 | 單一屬性滿點 | 進攻型，全剋+30% |
| 相生配置 | 相鄰屬性 | 平衡型，部分互相抵消 |
| 相剋配置 | 相剋屬性 | 輸出型，混合效果 |
| 無關配置 | 無直接關聯 | 防禦型，互相抵消 |
| 多餘屬性 | 合計>10 | 進攻型，超出部分單獨計算 |
| 全五行 | 四種以上 | 防禦型，抵消對方屬性 |

```
- 受敵傷害：-20%

如果對方也是不死系，反擊時：
- 被全剋：對敵傷害 -20%，受敵傷害 +20%
```

# 五、法術攻擊公式

## 5.1 公式架構

### 5.1.1 完整公式

```
最終法術傷害 = 基礎值 × 係數A × (係數B + 係數C) × 五行加成 × 隨機波動 × 其他修正
```

- **最大值**：0.8（達到靈力門檻時）

- **高於門檻**：最大0.8

### 5.4.4 範例

```

Step 1：計算靈力門檵
門檵 = 103 + 20 × 5 = 203

Step 2：計算係數B

      = (100 × (0.709 + 0.005)) / 100
```

## 5.5 係數C：靈攻發揮率

### 5.5.1 公式

```
靈攻門檵 = 10 + 30 × (法術等級 - 1)

非騎乘：係數C = (100 × ((自身靈攻 - 靈攻門檵) / 法術等級 / 10 + 0.005)) / 100
         範圍：0 ~ 0.4

騎乘：   係數C = (100 × ((主人靈攻 - 靈攻門檵) / 法術等級 / 10 × 0.5 + 0.005)) / 100
         範圍：0 ~ 0.2
```

### 5.5.2 數值範圍

- **非騎乘**：0 ~ 0.4
- **騎乘**：0 ~ 0.2
- **最小值**：0（靈攻不足時）
- **最大值**：達到門檵時

### 5.5.3 門檵值表

| 法術等級 | 靈攻門檵 |
|---------|---------|
| Lv1 | 10 |
| Lv2 | 40 |

| Lv5 | 130 |
| Lv6 | 160 |

| Lv9 | 250 |

### 5.5.4 範例

```

Step 1：計算靈攻門檵
門檵 = 10 + 30 × (5 - 1) = 130

Step 2：計算係數C
係數C = (100 × ((200 - 130) / 5 / 10 + 0.005)) / 100

      = (100 × 1.405) / 100

```

## 5.6 五行加成

### 5.6.1 公式

```
法術五行值 = 0 ~ 100
（10格五行 = 100，1格五行 = 10）

五行加成計算：
- 每點法術五行克制對方五行 → +2%
- 僅有增益，無減傷（與魔力寶貝一致）
```

### 5.6.2 說明

- 法術本身有五行屬性（例如火球術為火屬性）
- 攻擊時，法術五行會與对方五行產生加成
- **重要**：法術只有增益效果，没有被剋減傷

### 5.6.3 範例

```
情境：火球術（火屬性）攻擊 木80、土20 的怪物

Step 1：分析五行
- 火球術：火屬性 100
- 怪物：木80、土20

Step 2：計算
- 火與木：無直接關聯 → 0%
- 火與土：無直接關聯 → 0%
- 火與水：火剋水，但怪物無水 → 0%

結果：火球術對木土混合怪物無五行加成
```

```
情境：木系的藤蔓術攻擊 金60、水40 的怪物

Step 1：分析五行
- 藤蔓術：木屬性 100
- 怪物：金60、水40

Step 2：計算
- 木與金：無直接關聯 → 0%
- 木與水：水生木 → 相生關係 → 0%

結果：無五行加成
```

```
情境：金屬劍氣攻擊 木60、土40 的怪物

Step 1：分析五行
- 金屬劍氣：金屬性 100
- 怪物：木60、土40

Step 2：計算
- 金與木：無直接關聯 → 0%
- 金與土：土生金 → 相生 → 0%

結果：無五行加成
```

### 5.8.1 最終公式

# 六、物理攻擊公式

## 6.1 公式架構

### 6.1.1 完整公式

```
物理傷害 = 基礎攻擊² / (基礎攻擊/3 + 防禦) × (1 + 五行修正) × (1 + 種族修正) × 波動 × 武器倍率
```

## 6.2 攻擊與防禦計算

### 6.2.1 基礎公式

```
攻擊方最終攻擊 = { [ ( 攻擊加成 + 技能攻擊加成 ) / 100 × 面板攻擊 + 面板攻擊 ] } []取整

防禦方最終防禦 = { [ ( 防禦加成 + 技能防禦加成 ) / 100 × 面板防禦 + 面板防禦 ] } []取整
```

### 6.2.2 說明

- 攻擊加成：裝備、buff提供的攻擊力百分比
- 技能攻擊加成：技能提供的攻擊力加成
- 面板攻擊：角色基礎攻擊
- []取整：計算後取整數

## 6.3 攻防衰減（240理論）

### 6.3.1 閾值設定

```
閾值：241（部分版本250）

IF 最終攻擊 > 241:
    最終攻擊 = [ ( 最終攻擊 - 241 ) × 0.3 + 241 ]

IF 最終防禦 > 241:
    最終防禦 = [ ( 最終防禦 - 241 ) × 0.3 + 241 ]
```

### 6.3.2 說明

- 超過241點後，只發揮30%的效益
- 目的是防止數值無限膨脹
- 確保遊戲長期平衡

### 6.3.3 範例

```
情境：角色面板攻擊 300

Step 1：判斷是否超過閾值
300 > 241 → 需要衰減計算

Step 2：計算最終攻擊
最終攻擊 = (300 - 241) × 0.3 + 241
         = 59 × 0.3 + 241
         = 17.7 + 241
         = 258（取整）

Step 3：結論
原始300攻擊，實際只發揮258攻擊
衰減率 = (300 - 258) / 300 = 14%
```

## 6.4 基礎傷害計算

### 6.4.1 公式

```
IF 最終防禦 >= 0:
    傷害 = 最終攻擊² / ( 最終攻擊 / 3 + 最終防禦 )

IF 最終防禦 < 0:
    傷害 = 最終攻擊² / ( 最終攻擊 / 3 ) - 最終防禦
```

### 6.4.2 說明

- 防禦越高，傷害遞減
- 防禦為負數時，會增加傷害
- 這是魔力寶貝經典的物理公式

### 6.4.3 範例

```
情境：攻擊方攻擊 200，防禦方防禦 100

Step 1：計算
傷害 = 200² / (200/3 + 100)
     = 40000 / (66.67 + 100)
     = 40000 / 166.67
     = 240

Step 2：結論
物理傷害 = 240
```

```
情境：攻擊方攻擊 200，防禦方防禦 300

Step 1：計算
傷害 = 200² / (200/3 + 300)
     = 40000 / (66.67 + 300)
     = 40000 / 366.67
     = 109

Step 2：結論
物理傷害 = 109（防禦越高，傷害越低）
```

## 6.5 五行與種族修正

### 6.5.1 公式

```
非暴擊傷害 = 基礎傷害 × (100 + 五行剋制 + 種族剋制) / 100 × 波動 × [各種倍率]
```

### 6.5.2 說明

- **五行剋制**：物理攻擊同樣受五行相剋影響
- **種族剋制**：物理攻擊受種族系統影響
- 計算順序：先計算基礎傷害，再乘以修正係數

### 6.5.3 範例

```
情境：攻擊200，防禦100，五行剋制+15%，種族剋制+10%

Step 1：計算基礎傷害
基礎傷害 = 240

Step 2：計算修正
總修正 = 1 + 0.15 + 0.10 = 1.25

Step 3：計算最終傷害
最終傷害 = 240 × 1.25 = 300
```

## 6.6 暴擊傷害

### 6.6.1 公式

```
暴擊額外傷害 = [ 防禦方面板攻擊 × 攻擊方等級 / 防禦方等級 / 2 ] []取整

暴擊傷害 = 暴擊額外傷害 + 非暴擊傷害
```

### 6.6.2 說明

- 暴擊額外傷害基於雙方等級差
- 等級越高，暴擊傷害越高
- 暴擊時基礎傷害也會乘以波動

### 6.6.3 範例

```
情境：攻擊方 Lv50，防禦方 Lv30，攻擊方面板攻擊200，防禦方面板防禦100

Step 1：計算非暴擊傷害
非暴擊傷害 = 240

Step 2：計算暴擊額外傷害
暴擊額外 = [100 × 50 / 30 / 2] = [166.67 / 2] = 83

Step 3：計算暴擊傷害
暴擊傷害 = 240 + 83 = 323

Step 4：結論
暴擊傷害比普通傷害高 83 點 (+34.6%)
```

## 6.7 武器類型倍率

### 6.7.1 倍率表

| 武器類型 | 倍率公式 | 說明 |

| 法杖 | 無武器倍率 | 依賴法術攻擊 |

### 6.7.2 說明

- 武器倍率影響普通攻擊
- 技能攻擊有獨立倍率
- 弓類武器可以進行多次攻擊

### 6.7.3 範例


## 6.8 合擊公式

### 6.8.1 公式

```
最終合擊傷害 = [
    { 傷害A × (1 - 聖盾折扣率) } × (1 + 合擊人數 × 0.1)
] + [
    { 傷害B × (1 - 聖盾折扣率) } × (1 + 合擊人數 × 0.1)
] + ...
[]取整，傷害最低1
```

### 6.8.2 說明

- 合擊人數越多，傷害加成越高
- 每增加一人，傷害 +10%
- 聖盾可以降低合擊傷害

### 6.8.3 範例

```
情境：3人合擊，每個基礎傷害100，無聖盾

Step 1：計算每人的加成
加成 = 1 + 3 × 0.1 = 1.3

Step 2：計算最終合擊傷害
合擊傷害 = 100 × 1.3 + 100 × 1.3 + 100 × 1.3
         = 130 + 130 + 130
         = 390

Step 3：結論
3人合擊 = 390，單人攻擊 = 100 × 3 = 300
合擊增益 = +30%
```

### 7.4.3 概率分布


## 7.5 計算流程

### 7.5.1 完整流程

```
Step 1：確定靈防
假設：目標靈防 = 280

Step 2：計算區間
區間 = [273, 288]（序號17）

Step 3：查表得基礎值
法術等級10，序號17 → 數字18

Step 4：隨機選擇
從10個數字中隨機選取一個

Step 5：得出減傷率
假設選中數字5 → 減傷26%

Step 6：計算最終傷害
最終傷害 = 原法術傷害 × (100 - 26)% = 原法術傷害 × 74%
```

### 7.5.2 範例

# 九、完整公式彙編

## 9.1 法術傷害公式

### 9.1.1 標準公式

```
最終法術傷害 = 基礎值 × 係數A × (係數B + 係數C) × (1 + 五行修正) × (1 + 種族修正) × 波動

其中：
- 基礎值：查表（Lv1~Lv10）
- 係數A：0.1 ~ 1.1（靈力比決定）
- 係數B：0 ~ 0.8（靈力發揮率）
- 係數C：0 ~ 0.4（靈攻發揮率）
- 五行修正：-0.3 ~ +0.3
- 種族修正：-0.2 ~ +0.2
- 波動：0.9 / 1.0 / 1.1
```

### 9.1.2 簡化公式

```
法術傷害 = 基礎值 × 靈力比係數 × (靈力發揮率 + 靈攻發揮率) × 倍率修正
```

## 9.2 物理傷害公式

### 9.2.1 標準公式

```
IF 攻擊 > 241:
    攻擊 = (攻擊 - 241) × 0.3 + 241

IF 防禦 > 241:
    防禦 = (防禦 - 241) × 0.3 + 241

基礎傷害 = 攻擊² / (攻擊/3 + 防禦)

最終物理傷害 = 基礎傷害 × (1 + 五行修正) × (1 + 種族修正) × 波動 × 武器倍率

暴擊傷害 = 基礎傷害 × 波動 + [防禦方ATK × 攻擊方Lv / 防禦方Lv / 2]
```

### 9.2.2 簡化公式

```
物理傷害 = 攻擊² / (攻擊/3 + 防禦) × 倍率修正
```

## 9.3 屬性修正公式

### 9.3.1 五行修正

```
五行修正 = 0.3 × Σ(攻擊方五行 × 對方被剋系數)
```

### 9.3.2 種族修正

```
種族修正 = 對應剋制表（+0.2 / +0.1 / 0 / -0.1 / -0.2）
```

## 9.4 靈防減傷公式

### 9.4.1 計算流程

```
區間 = floor(靈防 / 16)
基礎值 = 查表(法術等級, 區間)
減傷率 = 隨機(1~10)對應概率 × 基礎值 / 100
```

### 9.4.2 簡化版

```
靈防減傷 = 固定區間減傷（非隨機）
例如：靈防 > 200 → 減傷15%
```
