## 一、底層邏輯（請先閱讀並理解）

### 1.1 五行剋制關係
```
木剋土 → 火剋金 → 土剋水 → 金剋木 → 水剋火
```
→ 攻擊方使用被剋制屬性時，額外增加 `傷害 × (怪物被剋制加成% / 100)`

### 1.2 五維抗性計算
```
實際受到屬性傷害 = 技能基礎傷害 × (1 - 抗性% / 100)
抗性上限：90%
```

### 1.3 升級公式
```
屬性增加量 = floor(基礎增加量 × 五行加權 × 等級衰減係數)
等級衰減係數 = 1 - (Lv / 200)
```
- 木本命 → HP / 採集力
- 火本命 → 攻擊力 / 魔法攻擊
- 土本命 → 防禦力 / 承重力
- 金本命 → 命中力 / 鍛冶力
- 水本命 → MP / 精鍊力 / 魔法攻擊

### 1.4 生活系屬性
採集力 / 鍛冶力 / 承重力 / 精鍊力 / 尋寶力
各由對應五行比例決定加成倍率。

## 三、後台功能建製要求

### 3.1 魔物建製（最重要的起點）

在 `/admin/game` 新增「魔物建製」頁面：

**新增欄位佈局（順序對應 DB）：**
```
基本信息區
  └─ monster_code / 名稱 / 五行 / 等級 / 稀有度
戰鬥屬性區
  └─ HP / 攻擊 / 防禦 / 速度 / 命中力 / 魔法攻擊
抗性區
  └─ 抗木% / 抗火% / 抗土% / 抗金% / 抗水% / 被剋制加成%
技能區（MonsterSkill DB）
  └─ skill_id_1（可搜尋下拉）/ skill_id_2 / skill_id_3
  └─ skill_id_3_unlock_level（覺醒解鎖）
AI與成長
  └─ AI_level（1-5下拉）/ 成長率
掉落區（關聯 Item DB）
  └─ 掉落素材ID / 掉落機率% / 傳說掉落ID / 傳說掉落機率%
  └─ 天命技能線索（BOOL，世界Boss專用）
```

**列表頁**：顯示 monster_id / monster_code / 名稱 / 五行 / 等級 / 稀有度 / HP / 攻擊 / 技能數量
**搜尋過濾**：五行 / 稀有度 / 等級範圍

## 四、請主系統 Agent 做的事情

### 4.1 必做
1. 在 `/admin/game` 後台新增六個建製頁面（對應六個 DB）
2. 每個頁面有「列表、新增、編輯」功能
3. 欄位完全按照上面的 Schema 設計
4. 確保新增/編輯後，資料寫入對應的 DB

### 4.2 前台串接（核心邏輯）
1. **戰鬥結算**：使用角色屬性（DB-001）和怪物抗性（Monster DB）計算傷害
2. **升級影響**：升級時根據五行比例計算屬性加成
3. **掉落判定**：擊敗怪物後，根據掉落機率%隨機掉落素材
4. **成就判定**：定時檢查角色狀態，滿足條件後自動頒發成就
