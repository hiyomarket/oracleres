# GD-012：鍛造系統設計文檔

**版本**：v1.0
**最後更新**：2026-03-23
**負責人**：遊戲 Agent
**狀態**：`已確認`

---

## 一、設計核心理念

先提出我的看法：玩家在大地圖上採集了大量素材，如果只是用來賣錢或解任務，很快就會失去動力。鍛造系統是消耗這些素材的最佳出口，也是玩家提升戰鬥力的核心途徑。

再來執行你的提案：本系統設計為「配方解鎖 + 機率產出」的模式。玩家需要收集特定素材來鍛造裝備，而產出的裝備品質（普通/精良/稀有/傳說）則取決於玩家的「火屬性（鍛冶力）」與「金屬性（精煉力）」。這強化了命理屬性在生活玩法中的重要性。

---

## 二、鍛造基礎機制

### 2.1 鍛造條件
1. **地點**：必須在「自宅」的「煉製台」進行。
2. **配方**：必須先解鎖對應裝備的配方（透過任務獎勵、商人購買或怪物掉落獲得）。
3. **素材**：消耗配方指定的素材數量。
4. **費用**：每次鍛造需支付一定數量的「靈石」作為手續費。

### 2.2 屬性影響
- **火屬性（鍛冶力）**：影響鍛造的「成功率」。火屬性越高，失敗（產出廢料）的機率越低。
- **金屬性（精煉力）**：影響產出裝備的「品質」。金屬性越高，產出高稀有度裝備的機率越大。

---

## 三、裝備品質與機率模型

鍛造成功後，系統會根據玩家的金屬性（精煉力）進行品質判定。

### 3.1 基礎品質機率表 (金屬性 0-20)

| 產出品質 | 顏色標示 | 基礎機率 | 屬性加成範圍 |
| :--- | :--- | :--- | :--- |
| **普通** | 白色 | 70% | 基礎數值 |
| **精良** | 綠色 | 25% | 基礎數值 + 10% |
| **稀有** | 藍色 | 4% | 基礎數值 + 25%，附加 1 條隨機詞綴 |
| **傳說** | 金色 | 1% | 基礎數值 + 50%，附加 2 條隨機詞綴 |

### 3.2 金屬性加成公式
玩家的「金屬性」每提升 10 點：
- 稀有品質產出率 +2%（從普通品質的機率中扣除）
- 傳說品質產出率 +0.5%（從普通品質的機率中扣除）

> **高衝擊目標（命理連動）**：
> 若玩家今日流日為「偏財」，鍛造出稀有與傳說裝備的機率直接翻倍。這會促使玩家在特定日子集中消耗素材進行鍛造。

---

## 四、隨機詞綴系統

當鍛造出「稀有」或「傳說」品質的裝備時，會隨機附加額外的詞綴效果。

### 4.1 詞綴池範例

| 詞綴名稱 | 適用部位 | 效果說明 |
| :--- | :--- | :--- |
| **鋒利的** | 武器 | 物理攻擊力額外 +5% |
| **沉重的** | 武器 | 物理攻擊力 +10%，但速度 -5 |
| **輕盈的** | 鞋子/下衣 | 速度 +5，閃避率 +2% |
| **堅固的** | 頭盔/上衣 | 物理防禦力額外 +8% |
| **聚能的** | 法杖/護符 | 魔法攻擊力額外 +8% |
| **吸血的** | 武器 (傳說專屬) | 每次攻擊恢復造成傷害 5% 的 HP |

---

## 五、裝備強化與精煉 (未來擴充預留)

為避免系統初期過於複雜，v1.0 僅開放基礎鍛造。未來 v2.0 將開放：
1. **強化 (+1, +2...)**：消耗同名裝備或特定強化石，提升基礎數值。
2. **精煉（洗詞綴）**：消耗「五行精華」，重新隨機生成裝備的附加詞綴。

---

## 六、後台管理需求 (供 Game CMS 參考)

主系統在實作 Game CMS 時，需在 `game_items` 或獨立的 `game_recipes` 資料表中加入以下欄位：
- `recipe_id` (配方 ID)
- `target_item_id` (產出裝備 ID)
- `material_1_id`, `material_1_qty` (所需素材 1 及數量)
- `material_2_id`, `material_2_qty` (所需素材 2 及數量)
- `cost_stones` (鍛造所需靈石)
- `base_success_rate` (基礎成功率，預設 90%)
