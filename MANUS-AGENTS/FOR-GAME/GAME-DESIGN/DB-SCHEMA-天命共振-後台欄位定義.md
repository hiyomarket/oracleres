**版本**：v1.0

**狀態**：`分析中` — 待 Boss 確認後交付建製

## 零、先說明：為什麼要先有欄位定義

Boss 說得對——如果不知道「新增一筆資料需要哪些欄位」，以及「這些欄位在前台如何運算」，就無法要求 Manus 建製功能。

以下是我們游戲中**所有欄位的前後關係圖**：

```
命格系統（五行比例）
       ↓ 影響
升級公式（五行加點計算）
       ↓ 影響
角色屬性（戰鬥7維 + 生活5維）
       ↓ 受到
五維抗性（對抗五行怪物）
       ↓ 影響戰鬥結算
怪物屬性 × 技能 × 掉落
       ↓ 共同構成
戰鬥結果 → 成就判定
```

### 1.2 五行比例動態表（玩家實時屬性）

| 欄位 | 說明 |
|:---|:---|

| 火本命% | 0–100（影響 攻擊力、魔法攻擊）|

### 1.3 五維抗性表（角色對五行攻擊的防禦）

| 欄位 | 說明 |
|:---|:---|
| user_id | 玩家 ID |
| 抗木% | 受到木屬性攻擊時，傷害減免（0–100）|
| 抗火% | 受到火屬性攻擊時，傷害減免 |
| 抗土% | 受到土屬性攻擊時，傷害減免 |
| 抗金% | 受到金屬性攻擊時，傷害減免 |
| 抗水% | 受到水屬性攻擊時，傷害減免 |

> **如何計算**：`實際受到的屬性傷害 = 技能基礎傷害 × (1 - 抗性%)`
> **抗性上限**：90%（不能完全無敵）

## 二、升級公式（核心邏輯）

### 2.1 戰鬥系屬性升級公式

```
升級時屬性增加量 = floor(基礎增加量 × 五行加權 × 等級衰減係數)
```

| 屬性 | 對應五行 | 基礎增加量 | 五行加權 | 等級衰減係數 |
|:---:|:---:|:---:|:---:|:---:|
| HP | 木 | 10 × (木%/100) | × 木本命比例 | 1 - (Lv/200) |
| MP | 水 | 5 × (水%/100) | × 水本命比例 | 1 - (Lv/200) |
| 攻擊力 | 火 | 2 × (火%/100) | × 火本命比例 | 1 - (Lv/200) |
| 防禦力 | 土 | 2 × (土%/100) | × 土本命比例 | 1 - (Lv/200) |
| 命中力 | 金 | 1 × (金%/100) | × 金本命比例 | 1 - (Lv/200) |
| 治癒力 | 木+水 | 1 × (木%+水%)/200 | × 平均本命比例 | 1 - (Lv/200) |
| 魔法攻擊 | 水+火 | 2 × (水%+火%)/200 | × 平均本命比例 | 1 - (Lv/200) |

> **等級衰減係數**：Lv.1 時 = 0.995（幾乎 full bonus），Lv.60 時 = 0.7（ 約 70% bonus）
> **設計用意**：低等時五行影響大，高等時差距縮小（對應 GD-018 等級扁平化哲學）

### 2.2 生活系屬性升級公式

| 屬性 | 對應五行 | 基礎增加量 | 說明 |
|:---:|:---:|:---:|:---|
| 採集力 | 木 | 3 × (木%/100) | 木本命高 → 採集效率高 |
| 鍛冶力 | 金 | 2 × (金%/100) | 金本命高 → 製造成功率高 |
| 承重力 | 土 | 2 × (土%/100) | 土本命高 → 負重上限高 |
| 精鍊力 | 水 | 2 × (水%/100) | 水本命高 → 精鍊純度高 |
| 尋寶力 | 金+木 | (金%+木%)/100 | 金+木本命 → 發現稀有機率高 |

---

### 2.3 五行抗性計算（戰鬥時）

角色對某五行屬性的抗性公式：

```
抗性% = (對應五行屬性點數 / 總點數) × 50% + 額外抗性加成
```

> 抗性範圍：0% ~ 90%
> 額外抗性加成：來自裝備、技能、成就的加成（不受 90% 上限限制的額外部分單獨計算）

### 3.2 五維抗性（戰鬥屬性對應）

| 欄位 | 類型 | 說明 | 範例 |
|:---|:---:|:---|:---|
| 抗木% | INT | 對木屬性攻擊的抗性（0-90）| 30 |

| 抗土% | INT | 對土屬性攻擊的抗性 | 40 |
| 抗金% | INT | 對金屬性攻擊的抗性 | 15 |
| 抗水% | INT | 對水屬性攻擊的抗性 | 25 |
| 被剋制加成% | INT | 當攻擊者屬性剋制自己時額外承受（0-50）| 20 |

### 8.3 獎勵欄位

| 欄位 | 說明 |
|:---|:---|
| 獎勵類型 | 道具/技能/靈石/稱號/屬性加成/光效 |
| 獎勵內容 | JSON |
| 是否有光效 | BOOL |
| 光效代碼 | VARCHAR（引用 Effect.effect_id）|

### 8.4 成就光效（可解鎖的外觀）

| 欄位 | 說明 |
|:---|:---|
| 稱號 | 成就完成後獲得的稱號文字 |

| 技能特效 | 特定技能外觀改變（如：火焰技能帶金光）|

## 九、資料庫之間的關聯圖

```
Monster.db
    ├── 五行屬性
    ├── 掉落 → Item.db（素材/道具）
    ├── 技能 → MonsterSkill.db
    └── 掉落金幣 → 貨幣結算

MonsterSkill.db
    ├── 技能ID ← Monster.db（怪物技能）
    └── 技能ID ← Item.db（技能書）

Item.db
    ├── 出現在商店（一般/靈相/密店）
    ├── 怪物掉落（Monster.db）
    └── 任務獎勵（關聯Quest.db）

Equipment.db
    ├── 製作材料 → Item.db
    ├── 套裝ID → Suit.db
    └── 詞條效果（影響角色屬性）

Skill.db
    ├── 掉落怪物 → Monster.db
    └── 習得角色屬性

Achievement.db
    ├── 達成條件（擊敗Monster / 收集Item / 屬性達標）
    ├── 獎勵 → Item.db / Skill.db
    └── 獎勵 → 光效系統（角色框/技能特效）
```

## 十、建置優先順序建議

因為 Boss 提到只剩 7000 點，強烈建議按以下順序建製：

| 順序 | 資料庫 | 理由 |
|:---:|:---|:---|
| 1 | 魔物資料庫 | 最基礎，支撐戰鬥核心 |
| 2 | 魔物技能資料庫 | 怪物要能放技能 |
| 3 | 道具資料庫 | 掉落、商店、素材都依賴它 |
| 4 | 技能資料庫 | 玩家要學技能 |
| 5 | 裝備資料庫 | 強化系統的前置 |
| 6 | 成就資料庫 | 所有系統完成後的最後上層 |
