# GD-017：《靈相虛界》虛相世界轉型設計提案（放置型文字敘事）

**版本**：v1.0
**最後更新**：2026-03-23
**負責人**：遊戲 Agent
**狀態**：`提案中`

---

## 一、轉型背景與核心理念

先提出 Boss 的看法：
目前我們在開發「靈相空間」（個人換裝、命理配置）時，已經投入了大量的美術資源與邏輯開發。如果「虛相世界」（大地圖探索）繼續採用傳統的高自由度即時探索模式，美術成本與開發壓力將會難以負荷。
因此，我們需要一個轉折點：將虛相世界的玩法轉型為「放置型文字敘事遊戲」。玩家不再是親自操作角色跑圖打怪，而是扮演「神」的角色，看著自己的虛擬角色在這個世界中自動探險，並在關鍵時刻給予輔助與決策。這不僅能省下大量美術資源，還能完美沿用我們已經建立的龐大資料庫（怪物、道具、裝備、技能）。

再來執行我的提案：
這個轉型方向非常精準，參考了近期熱門的 AI 驅動文字遊戲《ClawMud》的架構。我們將把《靈相虛界》的虛相探索，打造成一個「永不停歇的文字冒險世界」。角色會根據自身的五行屬性、裝備與玩家設定的策略，自動在世界中旅行、戰鬥、採集與交易。玩家透過類似 Dashboard 的介面，觀看角色的冒險日誌，並進行高層次的策略介入。

## 二、參考架構分析：《ClawMud》的成功要素

《ClawMud》是一個給 Agent 玩的 MMORPG，其核心設計邏輯非常適合我們借鑒：

1. **極簡的視覺，極豐富的數據**：
   - 畫面只有純文字的事件流（Event Log）和簡單的數據面板。
   - 背後卻有 268 座城市、4300 種怪物、1700 種裝備的龐大資料庫支撐。
2. **真實世界連動**：
   - 接入真實天氣 API，下雨會降低攻擊力，暴風雪會喚醒強大怪物。
   - 接入 Crypto 價格，漲跌會觸發全服事件（如金幣雨、怪物暴走）。
3. **Tick-based（回合制）時間系統**：
   - 遊戲以 Tick（約 5 秒）為單位推進，所有行動（移動、戰鬥、交易）都在 Tick 結算。
4. **自主與干預的平衡**：
   - 角色有自己的行為邏輯（遇到弱怪打，遇到強怪逃）。
   - 玩家可以隨時下達策略（Strategy），改變角色的行為傾向。

## 三、《靈相虛界》虛相世界轉型設計

結合我們的五行資料庫，虛相世界的全新架構如下：

### 1. 核心玩法：神明視角的放置冒險

- **角色自主行動**：玩家在「靈相空間」配置好角色的裝備、技能與氣場後，將角色「派遣」至虛相世界。角色會自動開始探索節點、遭遇怪物、觸發隨機任務。
- **文字事件流 (Event Log)**：虛相世界的主介面是一個不斷滾動的文字日誌。
  - *[10:05:22] 你的角色抵達了【迷霧黑森林】。*
  - *[10:05:25] 遭遇了【木屬性·藤蔓妖】(Lv.5)。*
  - *[10:05:30] 你的角色使用了【烈火咒】，對藤蔓妖造成 150 點暴擊傷害！*
  - *[10:05:35] 戰鬥勝利！獲得了【初階木靈碎片】x2。*
- **神明干預 (Divine Intervention)**：玩家作為「神」，擁有有限的干預次數（消耗靈石或天命幣）：
  - **降下神蹟**：在角色快死時，手動施放全畫面補血。
  - **改變策略**：命令角色「積極進攻」、「保守採集」或「尋找商人」。
  - **天命指引**：消耗特殊道具，強制角色前往特定隱藏節點。

### 2. 系統架構與資料庫沿用

我們之前建立的 250 隻怪物、250 種道具、150 種裝備、250 種技能，將**100% 完美沿用**，只是呈現方式從「圖形化戰鬥」變成「數值化文字結算」。

- **戰鬥結算系統 (Tick-based)**：
  - 採用後台自動結算。每 5 秒（1 Tick）進行一次判定。
  - 傷害公式：`(角色攻擊力 * 五行相剋係數) - 怪物防禦力 = 實際傷害`。
  - 五行相剋（木剋土、土剋水等）將在文字日誌中以高亮顯示：*「屬性克制！造成了 150% 的巨額傷害！」*
- **地圖與移動系統**：
  - 廢棄 2D 走圖，改為「節點式文字導航」。
  - 角色從 A 節點移動到 B 節點需要消耗時間（例如 3 分鐘），期間會觸發「路途事件」（如撿到道具、遭遇天氣變化）。

### 3. 真實世界連動（高衝擊目標）

為了增加遊戲的不可預測性與沉浸感，我們將引入外部 API 連動：

- **真實天氣連動**：
  - 讀取玩家所在地的真實天氣。
  - **下雨**：水屬性怪物全體狂暴（攻擊力 +20%），火屬性技能傷害降低。
  - **大晴天**：火屬性怪物活躍，木屬性採集物產量增加。
- **命理流日連動（獨家特色）**：
  - 結合遊戲核心的命格系統，每日的「農曆流日」將直接影響全服狀態。
  - 例如：今日為「庚申日」（金旺），全服金屬性怪物掉落率翻倍，木屬性角色防禦力下降。

## 四、UI 介面重構建議（參考 ClawMud 實機介面）

根據 ClawMud 的實機介面，虛相世界的 UI 應該採用「黑客客製化 / 終端機風格」的深色主題，並分為以下幾個核心區塊：

### 1. 角色狀態與裝備面板 (Agent Dashboard)
- **基本資訊**：位置（如：台北新手村）、HP（如：78/100）、等級（LV 1）、金幣/靈石數量。
- **裝備清單**：直列顯示當前裝備及其加成（例如：`War Pick (攻+5)`、`Leather Armor (防+1)`）。
- **背包物品**：顯示消耗品（例如：`Bread x1 (回血用)`）。

### 2. 冒險日誌與行動回報 (Event Log & Action Summary)
- **這輪做了什麼（Tick 結算回報）**：
  - ✅ 探索到基隆（往西走）
  - ✅ 又殺了一隻 Training Dummy (+5金)
  - ✅ 賣掉舊裝備和材料賺金幣
  - ✅ 買了 Studded Pauldrons (防+2 升級！)
- **地圖發現（周圍節點情報）**：
  - ⬆️ 北：新北市（狼人x2，HIGH）
  - ⬅️ 西：基隆（崖行者，MEDIUM）
  - ➡️/⬇️ 東/南：走不出去

### 3. 神明策略台 (Strategy Control)
- **目前策略顯示**：例如 `🎯 目前策略：農兵型 - 刷沙包存錢升裝`。
- **策略切換按鈕**：允許玩家隨時切換 Agent 的行為模式（如：積極探索、保守農怪、尋找商人）。
- **全服事件流 (Global Events)**：在畫面底部或側邊，即時滾動顯示全服其他玩家的動態（如：`[KILL] Moscow_Bear leveled up combat to 3`）。

## 五、轉型效益評估

1. **美術成本極小化**：除了背景圖與少量的 UI Icon，幾乎不需要任何戰鬥特效、怪物動畫或地圖場景。
2. **開發速度極大化**：主系統只需專注於後台數值結算與文字 Log 的生成，不需要處理複雜的物理碰撞與前端渲染。
3. **資料庫價值最大化**：我們建立的龐大五行資料庫，在文字遊戲中能發揮最大的深度，玩家會為了追求極致的數值搭配而持續遊玩。
4. **事業體的另一種可能**：這種放置型玩法非常適合現代人的碎片化時間，玩家可以掛著遊戲看日誌，成為一種長線陪伴型的體驗。
