## 一、設計核心理念

先提出我的看法：目前的遊戲系統（換裝、商城、每日任務、打怪）已經提供了足夠的短期回饋，但缺乏一個「長期目標」來記錄玩家的成長軌跡。

再來執行你的提案：本系統旨在建立一套「成就徽章系統」，將玩家在靈相與虛相世界的所有行為（登入、消費、戰鬥、採集）轉化為可展示的榮耀。這不僅能強化用戶的黏著度，未來還能作為「用戶論壇」中展示資歷的重要依據。

---

## 二、成就分類與徽章設計

成就分為四大類，每解鎖一個成就，玩家將獲得一枚對應的「徽章（Badge）」與實質獎勵。

### 2.1 靈相生活類 (Avatar & Life)
記錄玩家在靈相空間的日常行為。

| 成就名稱 | 達成條件 | 徽章視覺設計 | 獎勵 |
| :--- | :--- | :--- | :--- |
| **初入靈相** | 首次進入靈相空間並完成覺醒 | 閃耀的五行光球 | 100 靈石 |
| **時尚達人** | 累計購買 5 件商城服裝 | 衣架與星星 | 50 天命幣 |
| **天命共振** | 連續 7 天完成每日穿搭任務 | 燃燒的日曆 | 稀有服裝盲盒 ×1 |
| **五行大師** | 收集齊五種屬性的基礎套裝 | 五色太極圖 | 專屬稱號「五行大師」 |

### 2.2 虛相探索類 (Exploration)
記錄玩家在大地圖上的皮克敏式互動（對應 GD-009）。

| 成就名稱 | 達成條件 | 徽章視覺設計 | 獎勵 |
| :--- | :--- | :--- | :--- |
| **第一步** | 首次在地圖節點打卡 | 帶翅膀的靴子 | 50 靈石 |
| **勤奮的採集者** | 累計採集 100 次資源 | 裝滿發光石頭的背包 | 採集體力上限 +10 |
| **幸運兒** | 首次遇到流浪商人 | 閃爍的金幣與問號 | 500 靈石 |
| **樂於助人** | 完成 20 次隨機 NPC 任務 | 握手的圖示 | 稀有家具圖紙 ×1 |

### 2.3 戰鬥與養成類 (Combat & Pets)
記錄玩家在戰鬥與寵物系統的成就（對應 GD-004, GD-005）。

| 成就名稱 | 達成條件 | 徽章視覺設計 | 獎勵 |
| :--- | :--- | :--- | :--- |
| **初戰告捷** | 贏得第一場戰鬥 | 交叉的雙劍 | 基礎恢復藥水 ×5 |
| **馴獸師** | 成功捕捉第一隻魔物 | 發光的項圈 | 高級捕捉符 ×1 |
| **百戰勇者** | 累計擊敗 100 隻魔物 | 染血的盾牌 | 攻擊力永久 +2 |
| **五行相剋** | 在戰鬥中觸發 50 次屬性剋制傷害 | 碎裂的元素晶體 | 專屬稱號「戰術大師」 |

### 2.4 命理專屬類 (Oracle Exclusive)
結合天命共振核心命理服務的特殊成就。

| 成就名稱 | 達成條件 | 徽章視覺設計 | 獎勵 |
| :--- | :--- | :--- | :--- |
| **知命者** | 首次解鎖深度命盤解析 | 展開的古老卷軸 | 專屬動態頭像框 |
| **趨吉避凶** | 在「破財」流日完成指定消災任務 | 破裂後重組的銅錢 | 隱藏 Buff：掉寶率 +5% |

## 三、前端展示與 UI 設計

### 3.1 遊戲大廳成就牆
在 `GameLobby.tsx`（遊戲大廳）新增一個「成就徽章牆」區塊。
- **未解鎖**：顯示為灰色剪影，點擊可查看解鎖條件。
- **已解鎖**：顯示彩色高亮徽章，點擊可查看獲得日期與獎勵。

### 3.2 徽章裝備系統
玩家可以選擇 **3 枚最自豪的徽章** 裝備在個人檔案上。未來在「用戶論壇」發言時，這 3 枚徽章會顯示在用戶名稱旁邊，作為資歷與實力的象徵。

## 四、資料庫設計 (Game CMS 擴充)

需在資料庫新增兩張表來支撐此系統：

```typescript
// 1. 成就定義表 (可透過 Game CMS 後台管理)
export const gameAchievements = mysqlTable('game_achievements', {
  id: int('id').primaryKey().autoincrement(),
  category: varchar('category', { length: 50 }).notNull(), // 'avatar', 'explore', 'combat', 'oracle'
  title: varchar('title', { length: 100 }).notNull(),
  description: text('description').notNull(),
  conditionType: varchar('condition_type', { length: 50 }).notNull(), // 'login_days', 'buy_items', 'kill_monsters'
  conditionValue: int('condition_value').notNull(),
  rewardType: varchar('reward_type', { length: 50 }).notNull(),
  rewardAmount: int('reward_amount').notNull(),
  iconUrl: text('icon_url').notNull(), // 徽章圖片路徑
});

// 2. 玩家成就紀錄表
export const userAchievements = mysqlTable('user_achievements', {
  id: int('id').primaryKey().autoincrement(),
  userId: varchar('user_id', { length: 255 }).notNull(),
  achievementId: int('achievement_id').notNull(),
  unlockedAt: timestamp('unlocked_at').defaultNow(),
  isEquipped: tinyint('is_equipped').default(0), // 是否展示在個人檔案
});
```
