# UI-001：《靈相虛界》大廳介面與進入流程重設計提案

**版本**：v2.0
**最後更新**：2026-03-23
**負責人**：遊戲 Agent
**狀態**：`待主系統實作`

---

## 一、現有介面問題分析與核心邏輯重申

先提出 Boss 的看法：
目前的遊戲大廳介面太過粗糙，資訊全部擠在一個畫面裡，缺乏層次感。更嚴重的是，主系統似乎忽略了兩個最核心的邏輯：
1. **性別來源**：玩家在命格系統填寫資料時就已經有性別了，不需要重新判斷或讓玩家選。
2. **能力值來源**：這個遊戲是抓取玩家個人命格的「五行 % 數」來形成角色的初始能力值，命格系統是所有玩法的基地，不能想到什麼補什麼。
此外，大廳的設計應該參考《皮克敏 Bloom》的角色主頁面，以全螢幕角色展示為主，而不是把所有按鈕隨便丟在一起。

再來執行我的提案：
為了解決上述問題，我們需要重新梳理玩家進入《靈相虛界》的完整流程，強制主系統與命格系統進行深度資料串接，並以「皮克敏風格」重構靈相角色主頁面。

---

## 二、進入遊戲的完整流程與資料串接

玩家從主應用進入《靈相虛界》時，必須遵循以下流程，確保世界觀的沉浸感與邏輯性：

### 1. 首次進入（資料自動抓取與角色生成）
- **攔截跳轉**：若玩家尚未創建遊戲角色，點擊「進入遊戲」後，跳轉至「靈相覺醒」動畫頁面。
- **性別與立繪自動綁定**：
  - 系統**必須直接讀取**命格系統中的性別資料。
  - 根據性別與本命五行，自動掛載對應的初始立繪（TASK-008 產出的真實立繪）。
- **五行 % 數轉化能力值（核心邏輯）**：
  - 系統讀取玩家八字的五行比例（例如：木 40%、火 20%、土 10%、金 20%、水 10%）。
  - 將這些比例直接轉化為角色的初始基礎屬性：
    - **木 %** → 影響初始 HP 與恢復力
    - **火 %** → 影響初始攻擊力與暴擊率
    - **土 %** → 影響初始防禦力與護盾值
    - **金 %** → 影響初始速度與閃避率
    - **水 %** → 影響初始 MP 與魔法防禦
- **確認覺醒**：展示轉化後的能力值雷達圖，玩家確認後，正式進入靈相主頁面。

### 2. 日常進入（已創建角色）
- **登入動畫**：顯示簡短的五行流轉動畫（約 1.5 秒），帶出《靈相虛界》Logo。
- **進入主頁面**：直接進入重新設計後的「靈相角色主頁面」。

## 三、靈相角色主頁面 (Game Lobby) 重設計規範

參考《皮克敏 Bloom》的設計語彙，大廳不應該是一個「按鈕集合頁」，而應該是玩家在《靈相虛界》中的「個人樞紐」，強調角色與環境的融合。

### 1. 全螢幕沉浸式背景與角色展示 (Hero Section)
- **佔比**：佔據畫面上半部約 50%-60% 的空間。
- **背景**：取消單調的深色漸層，改用帶有五行流轉光暈的動態背景，或是根據玩家本命五行顯示專屬的自然環境（如木屬性為森林、水屬性為湖畔）。
- **真實立繪**：將 TASK-008 產出的高解析度立繪圖片置中放大顯示，角色腳下帶有對應本命五行的光暈特效。
- **互動**：點擊角色立繪，可觸發簡單的語音或文字氣泡（例如：「今日流日為水，宜向北行。」）。

### 2. 懸浮式頂部資訊 (Header)
- **左側**：玩家頭像與暱稱。
- **右側**：設定按鈕與資源欄位（天命幣、靈石），設計為半透明的懸浮膠囊，不遮擋背景。

### 3. 底部資訊卡片 (Bottom Cards)
採用類似皮克敏的圓角卡片設計，由下往上堆疊，可滑動展開。

#### 卡片一：個人名片與升級進度
- **頂部**：顯示玩家等級徽章、暱稱、帳號 ID。
- **進度條**：顯示「下一個升級的目標」，例如：
  - 收集 5 件木屬性裝備 (進度條)
  - 擊敗 10 隻初階怪物 (打勾狀態)

#### 卡片二：靈相空間與日常 (Personal Space)
這是玩家「先完成個人配置」的核心區域。
- **靈相空間入口**：點擊進入換裝、氣場配置與每日穿搭任務。
- **狀態提示**：若今日尚未完成穿搭任務，需有醒目的紅點或提示文字。

#### 卡片三：虛相探索與擴展 (World Exploration)
這是玩家配置完畢後，前往「探險」的區域。
- **虛相探索入口**：點擊進入大地圖、戰鬥與採集系統。
- **天命商城入口**：點擊進入商城購買靈石、服裝與道具。

## 四、給主系統的具體實作指令

請主系統開發者依照以下指令進行修改：

1. **強制串接命格系統資料（P0）**：
   - 廢棄目前的性別判斷邏輯，直接從命格系統讀取玩家性別。
   - 實作「五行 % 數轉化能力值」的演算法，並在角色創建時寫入 `game_users` 資料表。
2. **掛載真實立繪（P0）**：
   - 將立繪圖片路徑指向 S3 上 TASK-008 產出的真實圖片，替換掉目前的佔位圖。
3. **重構大廳佈局為「皮克敏風格」（P1）**：
   - 廢棄目前的單欄式列表佈局。
   - 實作「全螢幕背景 + 置中大立繪 + 底部圓角卡片堆疊」的沉浸式佈局。
4. **實作攔截與創建流程（P2）**：
   - 在進入 `/game` 路由前，檢查 `game_users` 表中是否有該玩家資料。
   - 若無，強制導向「靈相覺醒」頁面，展示五行轉化能力值的過程。
