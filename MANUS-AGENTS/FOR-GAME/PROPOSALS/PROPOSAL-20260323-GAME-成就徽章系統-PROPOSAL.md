## 遊戲功能說明 [必填]

### 功能名稱
成就徽章系統 (Achievement Badge System)

### 功能描述
先提出我的看法：目前玩家在靈相與虛相世界的行為（如換裝、打怪）缺乏長期紀錄，玩家無法向他人展示自己的「資歷」。

再來執行你的提案：本提案將實作一套成就系統。當玩家達成特定條件（如：首次完成每日任務、購買 5 件服裝）時，系統會自動發放對應的「徽章」與獎勵。玩家可以在遊戲大廳的「成就牆」查看已解鎖的徽章，並選擇 3 枚裝備在個人檔案上。

## 技術規格 [必填]

### 影響的現有檔案
| 檔案路徑 | 修改內容說明 |
|----------|-------------|
| `client/src/pages/game/GameLobby.tsx` | 新增「成就徽章牆」區塊 |
| `server/routers/gameAvatar.ts` | 在 `submitDailyAura` 中加入成就進度檢查 |
| `server/routers/gameShop.ts` | 在 `purchaseItem` 中加入成就進度檢查 |

### 新增的檔案
| 檔案路徑 | 用途說明 |
|----------|---------|
| `server/routers/gameAchievements.ts` | 處理成就查詢與裝備徽章的 API |
| `client/src/components/game/AchievementBadge.tsx` | 單一徽章的 UI 元件（支援 tooltip 顯示解鎖條件） |

### 資料庫異動 [必填]
需建立兩張資料表（詳細 Schema 請參考 GD-010）：
1. `game_achievements`：成就定義表（可由 Game CMS 管理）。
2. `user_achievements`：玩家成就紀錄表。
