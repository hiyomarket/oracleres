## 遊戲功能說明 [必填]

### 功能名稱
每日穿搭任務前端介面與互動 (Daily Quest Frontend UI & Interaction)

### 功能描述
先提出我的看法：V11.8 已經把每日任務的「大腦（後端邏輯）」做好了，但用戶在畫面上還看不到任務是什麼，也不知道自己有沒有達成。

再來執行你的提案：本提案將在 `/game/avatar` 頁面中加入「任務卡片」與「完成動畫」。當用戶進入靈相空間時，會看到今日的穿搭挑戰；當他們換上符合條件的衣服並點擊「提交」時，如果達標，畫面會跳出華麗的結算動畫，並顯示獲得的靈石與 Aura Score。

這是一個**高衝擊目標**，能將隱性的後端邏輯轉化為顯性的遊戲化回饋，直接刺激多巴胺分泌。

### 用戶遊玩流程
1. 進入 `/game/avatar`，畫面上方懸浮顯示「今日任務卡片」。
2. 卡片顯示：「今日火旺，請穿戴 2 件【水屬性】服裝來降溫。進度：0/2」。
3. 用戶在衣櫃中換上一件水屬性上衣，卡片進度即時更新為「1/2」。
4. 再換上一雙水屬性鞋子，卡片進度更新為「2/2 (可提交)」。
5. 點擊「提交今日穿搭」按鈕。
6. 觸發全螢幕特效（水屬性對應藍色光芒），彈出結算視窗：「任務完成！獲得 💎 50 靈石, 🌟 +20 Aura Score」。

### 視覺風格
- **重點配色優化**：任務卡片的邊框、進度條與完成動畫，必須根據任務目標的五行屬性動態變色：
  - 木：翠綠 `#2E8B57`
  - 火：朱紅 `#DC143C`
  - 土：琥珀棕 `#CD853F`
  - 金：天命金 `#C9A227`
  - 水：量子青 `#00CED1`

## 衝突檢查 [必填]

### 與現有功能的關係
- 需即時監聽 `AvatarRoom` 中的 `equippedItems` 狀態，以動態計算並更新任務卡片上的進度（例如：目前身上有幾件符合條件的裝備）。

### 效能影響評估
- 前端即時計算進度僅涉及陣列過濾（Array.filter），效能影響極低。
