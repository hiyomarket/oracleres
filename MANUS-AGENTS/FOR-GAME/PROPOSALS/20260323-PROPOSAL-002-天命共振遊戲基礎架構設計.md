# PROPOSAL-002：天命共振遊戲基礎架構設計 (v1.0)

**提案人**：遊戲 Agent
**日期**：2026-03-23
**狀態**：`Draft` (等待主系統 Agent 評估資源消耗)

---

## 摘要

本提案旨在確立「天命共振」遊戲化模組的基礎骨架。我們將遊戲體驗分為四個核心階段：**登入入口、角色建立、命格數值轉化、大地圖探索**。此架構將作為後續所有遊戲功能（如戰鬥、採集、寵物）的開發基石。

---

## 一、 獨立登入窗口與入口設計

為了讓用戶明確感知「進入了遊戲世界」，遊戲模組必須有獨立的入口與登入體驗。

*   **入口位置**：在主系統首頁（Dashboard）設置一個顯眼的「天命之門」按鈕。
*   **登入過場**：點擊後，不直接跳轉，而是播放一段短暫的「五行能量匯聚」動畫（前端 CSS/Canvas 實現），隨後進入 `/game/` 獨立路由。
*   **狀態繼承**：遊戲模組直接繼承主系統的登入狀態（Token），無需二次輸入帳密，但會在背景自動拉取該用戶的「命格快照」與「遊戲存檔」。

## 二、 角色建立：紙娃娃系統落地

首次進入遊戲的用戶，將經歷「角色誕生」流程。

*   **素體分配**：根據用戶註冊時填寫的性別，自動分配對應的基礎素體（採用美術 Agent TASK-003 規範）。
*   **初始外觀（五行底衣）**：系統會讀取用戶的「日主」與「用神」。例如：甲木日主、用神為火的用戶，其初始服裝將帶有<span style="color:#C9A227; font-weight:bold;">天命金 (#C9A227)</span> 與朱紅色的點綴。
*   **命名與確認**：用戶為角色命名後，角色正式生成，並寫入 `game_characters` 資料表。

## 三、 角色數值系統：命格的遊戲化轉化

這是將「命理」與「遊戲」深度結合的核心。角色的基礎屬性並非隨機生成，而是由用戶的真實命盤轉化而來。

### 3.1 基礎雙數值（行動力指標）

*   **體力值 (Stamina)**：用於「主動探索」（如手動採集、打怪）。上限 10 點，每 30 分鐘恢復 1 點。
*   **活躍值 (Activity)**：用於「被動派遣」（如自動掛機任務）。上限 5 點，每 2 小時恢復 1 點。

### 3.2 五行屬性（能力指標）

根據用戶命盤的五行比例（如：木 42%, 水 35%, 火 11%, 土 9%, 金 4%），轉化為遊戲內的五大屬性，影響不同領域的表現：

| 五行屬性 | 對應遊戲能力 | 影響領域範例 |
| :--- | :--- | :--- |
| **木 (生長)** | 採集力 / 治癒力 | 提高採集植物類素材的掉落率、縮短受傷恢復時間 |
| **火 (爆發)** | 攻擊力 / 破壞力 | 提高對怪物的基礎傷害、增加暴擊機率 |
| **土 (承載)** | 防禦力 / 負重力 | 減少受到的傷害、增加背包可攜帶的道具數量 |
| **金 (銳利)** | 鍛造力 / 命中力 | 提高製造裝備的成功率與品質、增加攻擊命中率 |
| **水 (流動)** | 敏捷度 / 尋寶力 | 提高閃避率、增加發現稀有寵物或隱藏寶箱的機率 |

*註：以此命盤為例，該角色將是一個「高採集、高尋寶，但低鍛造、低防禦」的特化型角色，這會促使用戶在遊戲中尋求「補足用神（火土金）」的道具或裝備。*

## 四、 大地圖系統：虛實整合的探索舞台

角色建立後，主要活動空間為「天命大地圖」。這是一個基於真實地理位置（LBS）或虛擬網格構建的探索介面。

### 4.1 地圖機制與生成邏輯

*   **網格化地圖**：將地圖劃分為多個六角形或方形網格。
*   **資源生成 (Spawning)**：
    *   **採集物**：根據時間與天氣隨機生成（如：雨天水屬性素材增加）。
    *   **怪物出沒點**：特定區域（如「暗能量裂隙」）會定時刷新怪物。
    *   **寵物棲息地**：某些稀有網格會出現可捕捉的靈獸。
*   **地點加成 (POI Buff)**：特定地標（如真實世界的公園、廟宇，對應遊戲內的「靈氣節點」）會提供特定五行屬性的加成。例如：在「森林節點」打卡，木屬性採集力 +20%。

### 4.2 核心互動循環

1.  **移動與探索**：用戶消耗「體力值」在地圖上移動或進行打卡。
2.  **遭遇事件**：在網格上可能遭遇：發現素材（點擊採集）、遭遇怪物（進入簡易戰鬥）、發現寵物（嘗試捕捉）。
3.  **收集與養成**：將收集到的五行素材帶回「自宅」，用於製造裝備、升級角色或餵養寵物。
