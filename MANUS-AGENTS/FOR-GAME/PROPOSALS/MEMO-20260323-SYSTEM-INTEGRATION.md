# 《靈相虛界》雙系統串接規格書 (System Integration Spec)

**版本**：v1.0
**日期**：2026-03-23
**負責人**：遊戲 Agent (代表 Boss)
**發送對象**：主系統 Agent

---

## 一、架構總覽：雙系統分工

為了加速開發，《靈相虛界》將分為兩個獨立運作但互相串接的系統：

1. **主系統 (Main System)**：負責「靈相空間」。處理用戶註冊、命格計算、紙娃娃換裝、商城購買、靈石/天命幣錢包。
2. **遊戲引擎 (Game Engine)**：負責「虛相世界」。這是一個獨立的 Node.js Tick-based 引擎，負責處理放置型文字冒險的邏輯（打怪、掉落、移動）。

**核心原則**：主系統是「資料的源頭（Source of Truth）」，遊戲引擎是「資料的消耗者與產出者」。

---

## 二、角色關係定義：靈相 vs 虛相

這兩個系統中的「角色」是同一個實體，但表現形式不同：

| 屬性 | 靈相角色 (主系統) | 虛相角色 (遊戲引擎) | 串接邏輯 |
| :--- | :--- | :--- | :--- |
| **視覺表現** | 2D 紙娃娃立繪 (Pikmin 風格) | 純文字狀態面板 | 遊戲引擎不處理圖片，只顯示文字狀態 |
| **性別** | 讀取 `userProfiles.gender` | 影響部分文字描述（如：他/她） | 遊戲引擎需從主系統取得性別 |
| **基礎能力值** | 由命格五行 % 數決定 | 戰鬥結算的基礎數值 | **核心串接點**：主系統需提供五行轉換後的能力值 |
| **裝備** | 影響外觀與 Aura Score | 影響戰鬥數值 (攻/防/速) | 遊戲引擎讀取主系統的裝備清單，套用數值加成 |
| **背包/道具** | 存放於主系統資料庫 | 在遊戲引擎中消耗使用 | 遊戲引擎消耗道具時，需呼叫主系統 API 扣除 |

---

## 三、API 串接點與資料結構 (API Contracts)

遊戲引擎（虛相世界）在運作時，需要呼叫主系統的以下 API：

### 1. 取得角色狀態 (Get Agent Status)
- **觸發時機**：玩家進入虛相世界，或遊戲引擎初始化 Agent 時。
- **方向**：遊戲引擎 -> 主系統
- **預期回傳結構**：
```json
{
  "userId": "uuid-1234",
  "gender": "male",
  "baseStats": {
    "hp": 150,      // 來自木屬性 %
    "attack": 25,   // 來自火屬性 %
    "defense": 15,  // 來自土屬性 %
    "speed": 10,    // 來自金屬性 %
    "mp": 50        // 來自水屬性 %
  },
  "equippedItems": [
    { "itemId": "WPN-001", "type": "weapon", "stats": { "attack": 10 } },
    { "itemId": "ARM-001", "type": "armor", "stats": { "defense": 5 } }
  ]
}
```

### 2. 結算同步 (Sync Tick Results)
- **觸發時機**：遊戲引擎每隔一段時間（例如每 10 個 Tick，或角色死亡/升級時），將結果同步回主系統。
- **方向**：遊戲引擎 -> 主系統
- **傳遞資料**：
```json
{
  "userId": "uuid-1234",
  "events": [
    { "type": "kill", "monsterId": "MOB-001", "expGained": 50 },
    { "type": "loot", "itemId": "MAT-001", "quantity": 2 }
  ],
  "walletUpdate": {
    "goldDelta": 150,     // 賺取 150 金幣
    "spiritStoneDelta": 0 // 靈石不變
  }
}
```

### 3. 神明干預扣款 (Divine Intervention Charge)
- **觸發時機**：玩家在虛相世界點擊「神蹟補血」或「天命指引」時。
- **方向**：遊戲引擎 -> 主系統
- **傳遞資料**：扣除指定數量的靈石或天命幣。若主系統回傳餘額不足，遊戲引擎則阻擋該操作。
