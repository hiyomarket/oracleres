## 遊戲功能說明 [必填]

### 功能名稱
天命日常互動任務（習慣養成引擎）

### 功能描述
「天命日常互動任務」是一個輕量級的遊戲化系統，旨在將每日的命理建議轉化為具體、可執行的微型任務。系統會根據當日的五行能量狀態與用戶的個人命格（如用神、忌神），每日自動生成 3 個專屬的「命理任務」。

用戶完成這些任務後，可以在系統內進行「打卡」或「收集能量球」的互動，並獲得「天命幣」或「積分」作為獎勵。這不僅能提升用戶每日登入的動力，也為未來更複雜的「命格共振養成系統」（如虛擬形象升級、自宅佈置）奠定資源累積的基礎。

### 命理連結說明 [必填]
本功能並非隨機生成任務，而是深度綁定天命共振的核心命理引擎：
1.  **任務生成基於五行生剋**：若今日流日為「火旺」，且用戶的用神為「火」，系統會生成「順應火能量」的任務（例如：穿著紅色系衣物、進行創意發想）；若用戶忌「火」，則會生成「化解火煞」的任務（例如：多喝水、保持冷靜）。
2.  **與作戰室能量連動**：任務的難度與獎勵倍率，可與 `/war-room` 中的「今日能量等級（energyLevel）」掛鉤。例如在「大吉」之日，完成任務可獲得額外加成。
3.  **命格活化**：讓用戶透過實際行動（如飲食、穿搭、作息調整）來「補運」，將抽象的命理概念轉化為具體的生活實踐。

### 用戶遊玩流程
1.  **登入與領取**：用戶每日登入系統，進入 `/game/daily-quest` 頁面（或透過首頁的懸浮入口進入）。
2.  **查看任務**：畫面上顯示今日專屬的 3 個命理任務，並附帶簡短的命理解釋（例如：「今日木旺，適合向外拓展，去認識一位新朋友吧！」）。
3.  **執行與互動**：用戶在現實生活中完成任務後，回到頁面點擊「完成」按鈕。畫面上會出現對應五行顏色的「能量球」動畫。
4.  **收集獎勵**：點擊能量球，系統發放天命幣/積分，並記錄於用戶的能量日誌中。
5.  **進度累積**：頁面上方顯示本週的「能量累積進度條」，連續完成可獲得額外寶箱獎勵。

### 資料庫異動 [必填]
> 所有資料表必須使用 `game_` 前綴

```typescript
import { int, mysqlTable, varchar, timestamp, tinyint, json } from "drizzle-orm/mysql-core";

// 記錄用戶每日生成的任務清單與狀態
export const gameDailyQuests = mysqlTable('game_daily_quests', {
  id: int('id').primaryKey().autoincrement(),
  userId: int('userId').notNull(),
  // 任務日期 (YYYY-MM-DD)
  dateString: varchar('dateString', { length: 12 }).notNull(),
  // 任務內容 (JSON 陣列，包含任務 ID、描述、對應五行、獎勵數量、是否完成)
  // 例: [{ questId: "q_fire_01", desc: "喝一杯熱咖啡", element: "火", reward: 10, isCompleted: false }]
  quests: json('quests').$type<Array<{
    questId: string;
    desc: string;
    element: string;
    reward: number;
    isCompleted: boolean;
  }>>().notNull(),
  // 當日是否已領取全勤寶箱
  isAllCompleted: tinyint('isAllCompleted').default(0).notNull(),
  createdAt: timestamp('createdAt').defaultNow().notNull(),
  updatedAt: timestamp('updatedAt').defaultNow().onUpdateNow().notNull(),
});

// 記錄用戶累積的五行能量（為未來的虛擬形象與農場做準備）
export const gameUserEnergyLogs = mysqlTable('game_user_energy_logs', {
  id: int('id').primaryKey().autoincrement(),
  userId: int('userId').notNull(),
  // 累積的五行能量值
  woodEnergy: int('woodEnergy').default(0).notNull(),
  fireEnergy: int('fireEnergy').default(0).notNull(),
  earthEnergy: int('earthEnergy').default(0).notNull(),
  metalEnergy: int('metalEnergy').default(0).notNull(),
  waterEnergy: int('waterEnergy').default(0).notNull(),
  updatedAt: timestamp('updatedAt').defaultNow().onUpdateNow().notNull(),
});
```

### 命理視覺元素
*   **五行色彩編碼**：任務卡片的邊框或背景會根據對應的五行（木綠、火紅、土黃、金白、水黑）進行微調。
*   **動態能量球**：當用戶點擊「完成」時，會彈出對應五行顏色的光球動畫，飛入上方的進度條中，強化「收集能量」的視覺回饋。
*   **重點配色優化**：關鍵的命理提示（如「你的用神！」）將採用系統預設的重點配色（如金色或亮橘色）進行高亮，確保視覺強調的一致性。

## 衝突檢查 [必填]

### 與現有功能的關係
*   **完全獨立**：本功能建構於 `/game/daily-quest`，不會修改 `/war-room` 或 `/oracle` 的核心 UI。
*   **資料單向讀取**：任務生成邏輯僅「讀取」`userProfile` 與 `dailyEnergy`，不會修改命理核心數據。
*   **獎勵整合**：發放獎勵時，會呼叫現有的 `pointsTransactions` 或更新 `users.gameCoins`，完全遵循現有的經濟系統規範。

### 效能影響評估
*   **低影響**：任務生成（`questGenerator`）僅在用戶每日首次進入該頁面時執行一次，並將結果快取於 `game_daily_quests` 資料表中。後續讀取皆為簡單的 DB 查詢，不會對伺服器造成顯著負擔。
*   **無阻塞**：動畫效果皆在前端（Client-side）執行，不會影響頁面載入速度。
