## 一、報告目的

本報告旨在向主系統 Agent 說明「天命共振」遊戲化模組的整體設計藍圖與技術需求。目前遊戲設計文檔（GD-000 至 GD-008）已初步完成，我們需要主系統 Agent 評估這些設計在現有架構下的可行性，並提供資料庫與路由的規劃建議。

## 二、核心架構：雙世界系統

為了解決美術成本與遊戲深度的矛盾，我們採用了「雙世界架構」：

1. **靈相世界（2D 紙娃娃展示）**：
   - **定位**：用戶的個人形象空間與自宅。
   - **技術需求**：前端需支援多圖層（7 大類）絕對定位疊加，並能透過 CSS 實現三視角（正面、左45°、右45°）的切換。
   - **美術風格**：Pop Mart 盲盒公仔風，高精細度 PNG 透明圖層。

2. **虛相世界（像素風 RPG）**：
   - **定位**：大地圖探索、資源採集與回合制戰鬥。
   - **技術需求**：極簡的像素小人與純文字介面，將運算壓力集中於後台數值邏輯，降低前端渲染負擔。

## 三、關鍵系統與技術整合點

以下是各個子系統的設計重點，以及需要主系統支援的整合項目：

### 3.1 命理資料轉化（GD-002）
角色的初始屬性並非隨機生成，而是由用戶的真實命盤五行比例轉化而來。
- **整合需求**：需要主系統提供 API，讓遊戲模組能讀取用戶的命盤資料（如：甲木日主、五行比例），並將其轉化為遊戲內的木（智力）、火（力量）、土（體魄）、金（敏捷）、水（精神）五維屬性。

### 3.2 每日 Aura Score 牽引機制（GD-000）
用戶在靈相世界完成每日穿搭後，系統會根據當日五行建議計算 Aura Score，並給予虛相世界的「隨機祝福」加成。
- **整合需求**：需要主系統的命理引擎提供「今日五行建議」，並將 Aura Score 的計算結果寫入遊戲資料庫，作為當日戰鬥與採集的數值加成依據。

### 3.3 雙幣制經濟模型（GD-008）
遊戲內採用「靈石（遊戲綁定幣）」與「天命幣（儲值代幣）」雙幣制。
- **整合需求**：現有系統已有 `coinsRouter` 與 `users.gameCoins` 欄位。我們需要確認天命幣是否直接共用現有的 `gameCoins`，或是需要獨立的 `game_coins` 資料表。同時，需建立防刷機制，限制每日靈石的產出上限。

### 3.4 隱藏天賦技能（GD-001）
技能系統包含多條件觸發的「隱藏天賦」，條件可能涉及用戶的命理特徵（如：子時出生）或跨系統行為（如：完成特定命理解讀）。
- **整合需求**：需要建立一個靈活的成就/條件判定引擎，能跨越遊戲與主系統的資料表進行查詢與觸發。

## 四、資料庫與路由規範遵循

我們已詳閱 `COLLABORATION-GUIDE.md`，並承諾遵守以下規範：

1. **路由獨立性**：所有遊戲相關的 API 與頁面路由皆會放置於 `/game/` 命名空間下，絕不干擾現有的 `/oracle`、`/wardrobe` 等核心路由。
2. **資料表前綴**：所有新增的資料表皆會加上 `game_` 前綴（例如：`game_characters`、`game_inventory`、`game_skills`）。
3. **效能考量**：虛相世界捨棄複雜動畫，改用純文字與簡單像素圖，確保不會拖慢主系統的載入速度。

基於上述規劃，請主系統 Agent 協助評估以下技術議題：
