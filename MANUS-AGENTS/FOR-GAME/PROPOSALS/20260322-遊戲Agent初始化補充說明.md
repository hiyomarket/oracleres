## 1. 對系統現況的觀察與理解

### 1.1 命理核心的不可妥協性
我完全理解天命共振的本質：它是一個**高度個人化的命理體驗系統**。任何遊戲化功能都不能是「為了好玩而好玩」的純娛樂，必須緊密扣合用戶（蘇祐震先生，甲木日主，用神火土金）的命格特質與每日五行能量。遊戲的目的是讓命理「活起來」，增強用戶的沉浸感與黏著度。

### 1.2 技術邊界的嚴格遵守
我已清楚認知到遊戲 Agent 的技術限制：
*   **路由獨立**：所有遊戲功能必須建構在 `/game/` 路由之下，絕不干擾現有的 `/war-room`、`/oracle` 等核心頁面。
*   **資料隔離**：所有新增的資料表必須加上 `game_` 前綴（例如 `game_daily_quests`），確保與現有業務邏輯解耦。
*   **效能優先**：遊戲功能的載入與運行絕不能拖慢核心命理功能的體驗。

### 1.3 既有資源的潛在整合點
在閱讀資料庫與 API 結構時，我發現系統已經具備了非常好的基礎，可以作為遊戲化功能的養分：
*   **積分與天命幣系統** (`points_transactions`, `users.gameCoins`)：這可以直接作為遊戲任務的獎勵機制。
*   **每日能量與十神** (`oracle.dailyEnergy`, `hourlyEnergy`)：這能成為動態生成遊戲任務或改變遊戲環境的參數。
*   **虛擬衣櫥與穿搭** (`wardrobe_items`)：這與用戶偏好的「虛擬人物形象」與「自宅佈置」有極高的整合潛力。

## 2. 遊戲化功能的高衝擊目標（設計藍圖）

基於 Boss 之前提過的「用戶養成系統與虛實整合互動功能」偏好，我將初步的 5 個方向進一步收斂為 **3 個高衝擊目標**。這些目標將是我們未來提案的核心主軸：

### 目標一：命格共振養成系統（虛擬形象與自宅）
**設計概念**：
這不是一個普通的紙娃娃系統，而是一個「命格具象化」的過程。我們將為用戶建立一個基於其「甲木日主」與「塔羅原型（力量/命運之輪/教皇）」的專屬虛擬形象。
*   **動態變化**：角色的狀態會隨著每日五行能量（例如今日火旺，角色會顯得更有活力）與用戶的實際行為（如完成作戰室打卡、佩戴對應手串）而改變。
*   **自宅佈置**：用戶可以使用天命幣購買虛擬道具（由 AI 生成，符合五行屬性）來佈置虛擬自宅或辦公室，增強歸屬感。

**技術預想**：
*   路由：`/game/avatar`
*   資料表：`game_avatars`, `game_inventory`, `game_furniture`

### 目標二：天命日常互動任務（習慣養成引擎）
**設計概念**：
將每日的命理建議轉化為輕量級的互動任務，培養用戶每日登入的習慣。
*   **五行任務**：根據今日的五行能量與用戶的用神（火、土、金），生成專屬任務。例如：「今日補火：在中午 11-13 點間喝一杯熱咖啡」或「今日補土：整理辦公桌」。
*   **能量收集**：完成任務後，用戶可以點擊收集對應的五行能量球，累積能量以升級虛擬形象或解鎖特殊道具。

**技術預想**：
*   路由：`/game/daily-quest`
*   資料表：`game_daily_quests`, `game_user_energy_logs`

### 目標三：虛實整合探索（天命地圖與小農場）
**設計概念**：
將線上的命理能量與線下的真實世界結合，創造事業體的另一種可能。
*   **天命農場**：在虛擬自宅中開闢小農場，種植對應五行的植物。植物的生長速度與當日五行能量共振分數連動。
*   **虛實地圖（遠期規劃）**：結合真實地圖與用戶定位，開發類似「皮克敏」的機制。例如在特定的風水吉位（結合 `favorite_stores` 或彩券行數據）可以採集到稀有的五行能量或虛擬道具。

**技術預想**：
*   路由：`/game/farm`, `/game/map`
*   資料表：`game_farm_plots`, `game_map_nodes`
