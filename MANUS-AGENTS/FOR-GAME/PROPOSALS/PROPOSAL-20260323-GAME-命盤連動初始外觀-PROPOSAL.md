## 遊戲功能說明 [必填]

### 功能名稱
命盤連動角色初始外觀生成 (Bazi-Linked Initial Avatar Generation)

### 功能描述
先提出我的看法：目前用戶進入靈相空間時，如果只有一個全裸的素體，體驗會非常乾癟，且無法體現「天命共振」的核心價值。我們需要讓用戶一進來就感受到「這個角色是為我而生的」。

再來執行你的提案：本提案將實作「初始外觀自動生成」機制。當用戶首次進入 `/game/avatar` 時，系統會讀取其八字命盤（日主五行），並自動從基礎服裝庫中，挑選一套符合其本命五行屬性的服裝（包含髮型、上衣、下衣、鞋子），免費發放並自動裝備。

這是一個**高衝擊目標**，能極大化用戶的首次「啊哈時刻（Aha Moment）」，讓他們立刻感受到命理與遊戲的深度結合。

### 命理連結說明 [必填]
初始服裝的分配邏輯完全基於用戶的「日主五行」（本命）：
- **木命人**：發放青綠色系、帶有自然元素的基礎套裝。
- **火命人**：發放紅紫色系、帶有熱情元素的基礎套裝。
- **土命人**：發放黃棕色系、帶有穩重元素的基礎套裝。
- **金命人**：發放白金色系、帶有俐落元素的基礎套裝。
- **水命人**：發放黑藍色系、帶有流動元素的基礎套裝。

### 用戶遊玩流程
1. 用戶首次點擊進入 `/game/avatar`。
2. 系統偵測到該用戶在 `game_wardrobe` 中沒有任何紀錄。
3. 系統呼叫命理引擎，取得用戶的日主五行。
4. 系統播放一段「天命共振，靈相覺醒」的短動畫。
5. 動畫結束後，角色穿著專屬的五行基礎套裝登場。
6. 系統彈出提示：「這是根據您的本命五行（X）為您生成的專屬靈相。」

### 資料庫異動 [必填]
無新增資料表。但需要在 `game_items`（由虛擬商城提案建立）中，預先設定好 5 套（對應五行）的「基礎套裝」，並標記為 `is_initial = 1`（需在商城提案的 Schema 中擴充此欄位，或透過特定 ID 範圍辨識）。

### 新的 tRPC Procedures
不需要新的 Procedure，而是擴充現有的 `getEquipped`：

```typescript
// 擴充邏輯草稿
getEquipped: protectedProcedure.query(async ({ ctx }) => {
  // 1. 查詢用戶當前裝備
  const equipped = await db.select().from(gameWardrobe).where(...);
  
  // 2. 如果有裝備，直接回傳
  if (equipped.length > 0) return equipped;
  
  // 3. 如果沒有裝備（首次進入），檢查是否擁有任何服裝
  const inventory = await db.select().from(gameWardrobe).where(...);
  if (inventory.length === 0) {
    // 4. 觸發初始生成邏輯
    const bazi = await getBazi(ctx.user.id); // 呼叫命理引擎
    const initialItems = getInitialItemsByWuxing(bazi.dayMasterWuxing);
    
    // 5. 寫入 game_wardrobe 並設定為裝備中
    await db.insert(gameWardrobe).values(initialItems);
    
    // 6. 回傳新生成的裝備，並帶上 isFirstTime 標記讓前端播動畫
    return { items: initialItems, isFirstTime: true };
  }
  
  return [];
})
```

## UI 設計說明 [必填]

### 版面配置
- **覺醒動畫**：全螢幕的五行光芒匯聚於中央素體，光芒顏色需對應生成的五行屬性。
- **提示彈窗**：
  ```text
  ┌─────────────────────────────┐
  │        靈相覺醒完成         │
  │                             │
  │ 根據您的八字命盤，您的本命  │
  │ 為【火】。已為您生成專屬的  │
  │ 緋紅套裝。                  │
  │                             │
  │        [ 開始共振 ]         │
  └─────────────────────────────┘
  ```

### 命理視覺元素
- **重點配色優化**：覺醒動畫的光芒與提示彈窗的按鈕，必須採用該用戶本命五行的**重點配色**（例如火命人使用紅色/金色），以強化個人化體驗。
