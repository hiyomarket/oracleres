## P0 修正一：魔法傷害精神比例公式

**修改檔案：** `server/tickEngine.ts`

### 問題
原本魔法係數 A 固定使用 `水命比例`，導致非水命玩家（如火命、木命）的魔法係數只有 0.1，魔法技能幾乎無效。

### 修正內容
新增 `calcSpiritCoeffA()` 函數，依據技能屬性（skillElement）對應玩家的五行數值：

| 技能屬性 | 使用玩家五行 | 對應敵方抗性 |
|:---:|:---:|:---:|
| 木 | wuxingWood | 金（metal）|
| 火 | wuxingFire | 水（water）|
| 土 | wuxingEarth | 木（wood）|
| 金 | wuxingMetal | 火（fire）|
| 水 | wuxingWater | 土（earth）|

係數 A 公式：`A = clamp(playerSkillStat / (enemyResistStat + 240), 0.1, 1.5)`

### 影響
- 火命玩家使用火技能時，係數 A 由 `水/240 ≈ 0.1` 提升為 `火/(敵方水+240)`
- 各命格玩家的魔法技能均能發揮應有威力
- `wuxingBoostDesc` 欄位記錄五行加成說明文字（如「火命施術，以火剋金」）

## P0 補充二：命盤五行換算公式統一

**修改檔案：** `server/tickEngine.ts`、`server/routers/gameWorld.ts`

### 問題
`gameWorld.ts` 的 `calcStatsFromNatal()` 使用舊公式（`HP = 100 + 木×2`），與 tickEngine 內部計算不一致，導致角色初始化時的屬性和戰鬥時的屬性有落差。

### 修正內容
在 `tickEngine.ts` 新增 `export function calcCharacterStats()`，作為全站統一公式：

```typescript
export function calcCharacterStats(
  natal: { wood: number; fire: number; earth: number; metal: number; water: number },
  level: number
) {
  return {
    hp:  Math.floor(100 + level * 10 + natal.wood  * 3.0),
    mp:  Math.floor(50  + level * 5  + natal.water * 2.0),
    atk: Math.floor(10  + level * 2  + natal.fire  * 1.5),
    def: Math.floor(8   + level * 1  + natal.earth * 1.2),
    spd: Math.floor(5   + level * 1  + natal.metal * 1.0),
  };
}
```

`gameWorld.ts` 的 `calcStatsFromNatal()` 改為呼叫此函數，確保三處（CharacterProfile、tickEngine、routers）使用同一套公式。
