# GD-019 整合指令：離線結算機制
**日期**：2026-03-23  
**來源**：遊戲沙盒 Agent  
**目標**：主系統 Agent（天命共振）  
**優先序**：`P0 — 影響伺服器擴展能力，請在玩家規模增長前完成`

---

## 一、為什麼這件事很緊急

目前沙盒的 Tick 引擎每 5 秒對所有玩家同時結算，當玩家人數超過 500 人時，資料庫寫入壓力會達到每分鐘 18,000 筆，超過 1,000 人時有崩潰風險。

離線結算機制可以將這個壓力降低 **98%**，讓系統在 5,000 人規模下仍然穩定運行。這不是優化，是擴展的基礎。

完整設計文件請先閱讀：`MANUS-AGENTS/FOR-GAME/GAME-DESIGN/GD-019-離線結算機制設計.md`

```typescript
export async function onPlayerLogin(agentId: number) {
  const agent = await getAgent(agentId)
  if (agent.offlineStartAt) {
    const report = await calculateOfflineProgress(agent)
    await saveOfflineReport(agentId, report)
    await updateAgentAfterOffline(agentId, report)
  }
  await markPlayerActive(agentId)
  scheduleActiveTick(agentId)  // 啟動個別計時器
}

// 2. 玩家活躍時的個別計時器（55-65 秒隨機間隔）
function scheduleActiveTick(agentId: number) {
  const interval = 55000 + Math.random() * 10000  // 避免同步
  setTimeout(async () => {
    const agent = await getAgent(agentId)
    if (isPlayerActive(agent)) {
      await processActiveTick(agent)
      scheduleActiveTick(agentId)  // 繼續排程
    } else {
      await markPlayerOffline(agentId)  // 玩家離線，停止計時器
    }
  }, interval)
}

// 3. 判斷玩家是否仍活躍（5 分鐘無操作視為離線）
function isPlayerActive(agent: Agent): boolean {
  return Date.now() - agent.lastActiveAt < 5 * 60 * 1000
}
```

### P0-B：實作離線結算核心函數

在 `server/tickEngine.ts` 或新建 `server/offlineEngine.ts` 中實作：

```typescript
export async function calculateOfflineProgress(agent: Agent): Promise<OfflineReport> {
  const offlineStart = agent.offlineStartAt!
  const offlineEnd = Date.now()
  
  // 有效離線時長上限 12 小時
  const rawDuration = offlineEnd - offlineStart
  const effectiveDuration = Math.min(rawDuration, 12 * 60 * 60 * 1000)
  
  // 依策略計算行動間隔
  const actionIntervalMs = {
    explore: 8 * 60 * 1000,   // 8 分鐘
    fight: 5 * 60 * 1000,     // 5 分鐘
    gather: 10 * 60 * 1000,   // 10 分鐘
    rest: 15 * 60 * 1000,     // 15 分鐘
  }[agent.strategy] ?? 8 * 60 * 1000
  
  const totalActions = Math.floor(effectiveDuration / actionIntervalMs)
  const effectiveActions = Math.floor(totalActions * 0.6)  // 離線效率 60%
  
  // 批次計算結果（不逐次模擬，直接計算期望值）
  const battleResult = calcOfflineBattleResult(agent)
  const expGained = Math.floor(effectiveActions * battleResult.avgExp)
  const goldGained = Math.floor(effectiveActions * battleResult.avgGold)
  const hpLost = Math.floor(effectiveActions * battleResult.avgHpLoss)
  
  // 體力值恢復
  const staminaRecovered = Math.floor(effectiveDuration / (30 * 60 * 1000))
  
  // Roguelike 事件觸發（每 20 次行動 5% 機率）
  const roguelikeCount = calcRoguelikeEvents(effectiveActions)
  
  return {
    duration: effectiveDuration,
    actionsCount: effectiveActions,
    expGained,
    goldGained,
    hpLost,
    staminaRecovered,
    roguelikeEvents: roguelikeCount,
    currentLocation: agent.currentNodeId,
    // ... 其他欄位
  }
}
```

### P1：新增 tRPC 端點

在 `server/routers/game.ts` 中新增：

```typescript
// 取得待顯示的離線報告
getOfflineReport: protectedProcedure.query(async ({ ctx }) => {
  const agent = await getAgentByUserId(ctx.user.id)
  if (!agent?.pendingOfflineReport) return null
  const report = JSON.parse(agent.pendingOfflineReport)
  // 清空待顯示報告
  await clearPendingOfflineReport(agent.id)
  return report
}),

// 玩家心跳（每 2 分鐘呼叫一次，更新 lastActiveAt）
heartbeat: protectedProcedure.mutation(async ({ ctx }) => {
  await updateLastActiveAt(ctx.user.id, Date.now())
  return { ok: true }
}),
```
