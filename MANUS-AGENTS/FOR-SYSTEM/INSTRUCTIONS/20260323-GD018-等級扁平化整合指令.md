# GD-018 整合指令：等級扁平化與核心成長系統
**日期**：2026-03-23  
**來源**：遊戲沙盒 Agent  
**目標**：主系統 Agent（天命共振）  
**狀態**：`請閱讀並整合` — 這份文件定義了整個遊戲的核心成長哲學，請在開發任何與角色成長相關的功能前先讀完

---

## 一、為什麼這份文件很重要

這份文件解決的是一個根本性的設計問題：**我們的遊戲不能變成單純刷等的計時器**。

目前沙盒的 Tick 引擎每 5 秒結算一次，升級速度過快，玩家很快就會發現「掛著不管就能升級」，整個命格連動的差異化就失去意義了。GD-018 提出的「等級扁平化」設計，是讓命格五行真正成為遊戲核心的關鍵。

請閱讀完整設計文件：`MANUS-AGENTS/FOR-GAME/GAME-DESIGN/GD-018-等級扁平化與核心成長設計.md`

## 二、需要你立即對齊的三個核心原則

### 原則 1：等級只是「通行證」

等級決定玩家能去哪些地圖節點，但**不直接決定戰力**。每升一級，屬性只增加極小量（HP +5、攻擊 +1），相比一套好裝備帶來的 +500 HP、+150 攻擊，等級的影響微乎其微。

**請你在靈相空間的角色面板中，把等級顯示降低視覺比重**，不要讓它看起來是最重要的數字。

### 原則 2：力量來自三個地方，不是等級

玩家的真實戰力來自：**裝備組合 × 技能搭配 × 寵物選擇**。這三個系統的設計細節都在 GD-018 文件中，包含：

- 裝備：5 色品質、8 欄位、套裝效果、三類詞條（基礎/特殊/命格）
- 技能：靠收集獲得（不靠升級），4 主動 + 2 被動，有 Combo 化學反應
- 寵物：個體值差異系統，三種用途（戰鬥/採集/命格共鳴）

### 原則 3：體力值控制節奏，不是等級

玩家每天的有效行動次數由**體力值**控制，而不是等級。這確保了：
- 高等級玩家不會無限刷資源
- 付費玩家的優勢有明確上限（每天最多補充 2 次活躍值）
- 每天的有限行動次數創造了「今天要做什麼」的策略選擇

## 三、需要你調整的現有功能

### 3.1 角色面板（靈相世界頁面）

目前的角色面板顯示：等級、HP/MP、攻防速、五行比例、金幣/靈石。

**請調整為以下優先序**（由上到下，視覺比重依序遞減）：
1. 旅人名稱 + 命格（本命五行）— 最大字體
2. HP/MP 狀態列 — 最顯眼
3. 裝備欄位（8 格，目前空槽顯示虛線框）— 中等比重
4. 技能欄位（4 主動 + 2 被動，目前空槽）— 中等比重
5. 五行屬性值（木/火/土/金/水 的實際數值）— 較小
6. 等級 + 經驗值 — **最小，放在角色面板底部**

### 3.2 Tick 引擎升級速度

沙盒目前的升級速度過快，請在整合時調整 EXP 獲取量，讓玩家在 Lv.1-10 需要約 1-3 天，而不是幾小時。

具體調整位置：`SANDBOX/oracle-mud/server/tickEngine.ts` 中的 `BASE_EXP_PER_FIGHT` 常數，建議從目前值降低至 1/5。

### 3.3 大廳改版（再次確認）

登入後的主畫面應該是靈相虛界的旅人狀態，不是功能卡片清單。旅人面板的第一眼應該讓玩家看到「我的旅人現在在哪裡、狀態如何、裝備欄有沒有空格需要填」，而不是等級數字。

## 四、需要你新增的 DB 欄位

以下欄位需要加入 `gameAgents` 表，才能支援 GD-018 的系統：

```sql
-- 體力值系統
stamina INT DEFAULT 10,          -- 活躍值（上限 10，每 30 分鐘 +1）
maxStamina INT DEFAULT 10,
staminaLastRegen BIGINT,         -- 上次恢復時間（Unix timestamp ms）

actionPoints INT DEFAULT 5,      -- 靈力值（上限 5，每 2 小時 +1）
maxActionPoints INT DEFAULT 5,
actionPointsLastRegen BIGINT,

-- 裝備欄位（8 格，存 JSON 或關聯表）
equippedHead TEXT,               -- 靈冠（道具 ID）
equippedBody TEXT,               -- 靈甲
equippedHands TEXT,              -- 靈手
equippedFeet TEXT,               -- 靈靴
equippedWeapon TEXT,             -- 靈器
equippedOffhand TEXT,            -- 靈盾/靈符
equippedRingA TEXT,              -- 靈環 A
equippedRingB TEXT,              -- 靈墜

-- 技能欄位（6 格）
skillSlot1 TEXT,                 -- 主動技能 1（技能 ID）
skillSlot2 TEXT,
skillSlot3 TEXT,
skillSlot4 TEXT,
passiveSlot1 TEXT,               -- 被動技能 1
passiveSlot2 TEXT,

-- 寵物欄位
activePetId TEXT,                -- 當前戰鬥寵物
gatheringPetId TEXT,             -- 當前採集寵物
spiritBeastId TEXT,              -- 靈獸（命格共鳴）

-- 角色命名
characterName TEXT,              -- 玩家自訂的旅人名稱（首次登入設定）
```

## 五、需要你新增的頁面/功能

### 5.1 首次登入命名畫面（P0，最優先）

玩家首次進入遊戲時，如果 `characterName` 為空，顯示命名畫面：
- 輸入旅人名稱（2-8 個字，不可使用特殊符號）
- 選擇性別（男/女，影響紙娃娃素體）
- 顯示命格五行分析（從主系統 API 取得的 `natalStats`）
- 確認後建立角色，不可更改名稱（或消耗天命幣更改）

### 5.2 裝備欄位 UI（P1）

在旅人 Tab 中，顯示 8 格裝備欄位：
- 空槽：顯示欄位圖示 + 「空」字樣，點擊可從背包選擇裝備
- 已裝備：顯示裝備名稱 + 品質顏色（灰/綠/藍/紫/橙）
- 長按可查看裝備詳情（詞條清單）

### 5.3 Roguelike 事件通知（P1）

當旅人觸發 Roguelike 事件時，在頂部顯示小紅點通知。玩家點擊後進入事件處理畫面，顯示事件描述 + 選項按鈕。未處理的事件最多存放 5 個。
