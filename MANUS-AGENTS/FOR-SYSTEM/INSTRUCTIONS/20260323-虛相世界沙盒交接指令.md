## 三、DB Schema 詳細說明

```sql
-- 角色表
gameAgents {
  id, userId, name, gender,
  level, exp,
  hp, maxHp, mp, maxMp,
  attack, defense, speed,
  wuxingWood, wuxingFire, wuxingEarth, wuxingMetal, wuxingWater,
  dominantElement,
  gold, spiritStones,          -- 金幣 / 靈石
  currentNodeId,               -- 當前地圖節點 ID
  strategy,                    -- explore / farm / gather / rest
  status,                      -- idle / moving / fighting / gathering / resting / dead
  createdAt, updatedAt
}

-- 事件日誌表
agentEvents {
  id, agentId, eventType,      -- move/combat/loot/gather/trade/rest/levelup/divine/death
  message,                     -- 文字描述
  data,                        -- JSON 附加資料（傷害數字、掉落物等）
  createdAt
}

-- 背包表
agentInventory {
  id, agentId, itemId, itemName, itemType, quantity, createdAt
}

-- 世界狀態表
gameWorld {
  id, tickCount,
  dailyStem, dailyBranch, dailyElement,  -- 今日天干地支流日
  activeAgents, totalTicks,
  updatedAt
}
```

## 六、關鍵設計決策（請遵守）

以下是沙盒開發過程中確認的設計原則，整合時請保持一致：

1. **大廳主畫面 = 靈相虛界**：用戶登入後直接看到旅人的即時狀態，不是功能卡片清單
2. **底部 Tab 導航**：靈相空間（換裝）和天命商城改為底部 Tab，不再是大廳的主體卡片
3. **神明視角**：玩家是「神」，旅人是「自動行動的角色」，玩家只做高層次干預（策略/神蹟）
4. **五行三維系統**：每個五行影響「角色素質 + 生活系 + 戰鬥系」三個維度（詳見 GD-002）
5. **體力值 = 行動限制**：活躍值（掛機消耗）控制自動行動頻率，體力值（手動消耗）控制主動操作次數
