# 五行屬性升級系統分析

## 現有系統分析

### 現有 calcCharacterStats 公式
```
hp   = 100 + level * 10 + wood  * 3.0
atk  = 10  + level * 2  + fire  * 1.5
def  = 10  + level * 1.5 + earth * 1.5
spd  = 5   + level * 0.5 + metal * 0.8
matk = 10  + level * 2   + water * 1.5
mp   = 50  + level * 5   + water * 1.5
```

### 現有五行屬性來源
- 初始值：全部 20（均勻分配）
- 成長方式：只有「注靈」行為才能增加五行屬性（每次 +0.5~2.0，上限 100）
- 升級時：**不會自動增加五行屬性**，只是用 calcCharacterStats 重新計算

### 問題
1. 五行屬性只能透過注靈增加，升級本身不會增加五行屬性
2. 等級對戰鬥數值的影響太線性（level * 10 HP, level * 2 ATK）
3. 五行屬性的加成係數太低（wood * 3 vs level * 10），等級主導一切
4. 沒有角色五行傾向的差異化成長

## 設計方案：五行屬性升級成長系統

### 核心概念（類似魔力寶貝）
- 木 → 體力（HP 成長）
- 火 → 力量（ATK 成長）
- 土 → 強度（DEF 成長）
- 金 → 速度（SPD 成長）
- 水 → 魔法（MP/MATK 成長）

### 升級時五行屬性自動成長
每次升級時，角色自動獲得 5 點五行屬性點數，分配規則：
1. 主屬性（dominantElement）獲得 2 點
2. 相生屬性獲得 1.5 點（木→火→土→金→水→木）
3. 其他三屬性共分 1.5 點（各 0.5 點）

### 五行屬性 → 戰鬥數值推導（新公式）
```
hp   = 80 + level * 8 + wood  * 5.0
atk  = 5  + level * 1 + fire  * 2.5
def  = 5  + level * 1 + earth * 2.5
spd  = 3  + level * 0.3 + metal * 1.5
matk = 5  + level * 1 + water * 2.5
mp   = 30 + level * 3 + water * 2.0
```

### 設計原則
- 降低等級的直接加成，提高五行屬性的影響力
- 讓角色的五行傾向真正影響戰鬥風格
- 升級 = 五行屬性成長 = 戰鬥數值成長，形成完整循環
