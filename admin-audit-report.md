## 二、六大管理領域詳細分析

### 2.1 核心管理（儀表板 + 用戶 + 客群）

**管理員儀表板** 是後台的入口，提供 KPI 數據卡片（用戶統計、天命幣流通）、功能使用頻率統計、方案訂閱分佈、24 小時活躍時段圖表等。整體功能完整，但缺少錯誤狀態的 UI 回饋。

**用戶管理** 是最常用的工具之一，支援搜尋、篩選（方案/生命靈數/活躍狀態/角色）、訂閱管理、積分贈送、命理師資格升降等。程式碼量達 1,232 行，但遊戲幣贈送功能尚有 TODO 標記未完成，訂閱管理彈窗中手動輸入模塊 ID 容易出錯。

**客群分組管理** 支援建立/編輯/刪除分組、批量調整成員方案與積分。功能完整但使用了原生 `confirm` 彈窗，與整體 UI 風格不一致。

### 2.2 商業營運（商業中心 + 功能兌換 + 天命小舖 + 行銷）

**商業中心** 是最大的營運工具（1,479 行），包含模塊管理、方案定價、行銷活動三大區塊。模塊選擇列表過長缺乏搜尋功能，部分程式碼使用 `as any` 型別轉換暗示結構不穩定。

**功能兌換中心管理** 處理功能方案的 CRUD 和訂單審核，功能完整但缺少骨架屏載入狀態。

**天命小舖管理** 設定 AI 功能費用和方案贈幣，但充值套餐為硬編碼顯示，尚未與金流串接。

**行銷中心** 管理 WBC 賽事和經濟系統配置，但賽程資料為寫死內容，部分功能已移除。

### 2.3 內容管理（廣告 + 主題 + 邏輯配置）

**全站廣告/公告管理** 支援 CRUD、排序、時間範圍設定，功能完整但日期選擇器使用原生元件，跨瀏覽器體驗不一致。

**全站主題管理** 提供主題預覽和套用，功能完整但預覽機制直接修改 DOM 可能產生副作用。

**邏輯計算權限** 管理能量模擬器規則、手串配飾、餐廳分類，頁面功能複雜（1,326 行）但缺少用戶引導說明。

### 2.4 專家系統

**專家管理** 涵蓋專家列表、預約訂單、命理師申請審核、用戶訊息、聯盟設定五個分頁。管理員備註功能在 UI 中存在但資料未送到後端，是一個需要修復的功能缺口。

### 2.5 安全存取

**特殊存取 Token 管理** 支援生成、管理、配置存取權杖，功能完整度高，但主列表缺乏分頁機制。

### 2.6 遊戲系統

**靈相世界 CMS** 是整個後台最龐大的頁面（5,366 行），包含圖鑑管理（魔物/道具/裝備/技能/成就/寵物）、商店管理（5 種商店）、AI 工具、戰鬥平衡、世界管理、系統管理六大區塊。部分區塊標示為 TODO，樣式不一致問題較多。

**遊戲劇院** 提供角色管理、全域參數、引擎監控等功能（2,531 行），角色編輯面板超過 30 個輸入框同時顯示，資訊過載嚴重。

## 三、跨頁面共通問題

經過完整掃描，以下是在多個頁面中反覆出現的共通問題：

| 問題類型 | 影響頁面數 | 嚴重程度 | 說明 |
|---------|-----------|---------|------|
| **佈局不統一** | 15/15 | 高 | 混用 DashboardLayout、AdminLayout、Custom 三種佈局，導航體驗碎片化 |
| **原生 confirm 彈窗** | 8/15 | 高 | 使用瀏覽器原生 confirm() 而非統一的 AlertDialog 元件 |
| **響應式設計缺失** | 12/15 | 高 | 多數頁面在手機或小螢幕上排版會亂掉 |
| **載入狀態不足** | 10/15 | 中 | 缺少骨架屏（Skeleton），僅顯示通用文字或完全空白 |
| **錯誤處理薄弱** | 11/15 | 中 | 後端請求失敗時缺乏明確的錯誤 UI 回饋 |
| **表單驗證不完整** | 9/15 | 中 | 缺少欄位級即時錯誤提示，僅依賴 toast 通知 |
| **無障礙性不足** | 13/15 | 中 | 缺少 ARIA 屬性、htmlFor 關聯、螢幕閱讀器支援 |
| **分頁機制缺失** | 6/15 | 中 | 大量資料一次載入，缺乏分頁或虛擬滾動 |
| **功能未完成** | 7/15 | 低 | 存在 TODO 標記或硬編碼佔位內容 |

## 四、優化方向建議

根據以上分析，我建議分三個階段進行優化，從「體感最明顯」到「長期穩固」：

### 第一階段：統一框架與導航（影響最大）

目前後台最大的問題是**佈局碎片化**——15 個頁面使用了三種不同的佈局方式，用戶在不同頁面之間切換時，導航位置、側邊欄、頂部列都不一致。

**具體做法：**
- 建立統一的 `AdminLayout` 組件，所有 `/admin/*` 頁面共用
- 左側固定側邊欄，按六大領域分組（核心管理 / 商業營運 / 內容管理 / 專家系統 / 安全存取 / 遊戲系統）
- 頂部列顯示當前頁面標題 + 麵包屑導航
- 側邊欄支援收合（手機端自動收合為漢堡選單）

### 第二階段：UI 品質提升（體驗最直接）

**2a. 統一互動元件**
- 將所有原生 `confirm()` 替換為 shadcn/ui 的 `AlertDialog`
- 將所有原生日期選擇器替換為統一的 `DatePicker` 元件
- 建立統一的 `DataTable` 元件（含排序、篩選、分頁）

**2b. 載入與錯誤狀態**
- 每個頁面加入骨架屏（Skeleton Loader）
- 建立統一的錯誤邊界（Error Boundary）和重試機制
- 表單加入欄位級即時驗證回饋

**2c. 響應式設計**
- 所有表格在手機端改為卡片式呈現
- 表單在小螢幕上改為單欄佈局
- 彈窗在手機端改為全螢幕抽屜

### 第三階段：功能補齊與整合

**3a. 修復未完成功能**
- 用戶管理：完成遊戲幣贈送功能
- 專家管理：修復管理員備註未送後端的問題
- 商業中心：修復 `as any` 型別轉換問題
- 天命小舖：充值套餐改為可配置（為金流串接做準備）

**3b. GameCMS 拆分**
- 靈相世界 CMS 目前 5,366 行集中在一個檔案，建議拆分為獨立的子頁面：
  - `/admin/game/catalogs` — 圖鑑管理
  - `/admin/game/shops` — 商店管理
  - `/admin/game/balance` — 戰鬥平衡
  - `/admin/game/world` — 世界管理
  - `/admin/game/settings` — 系統設定

**3c. 遊戲劇院簡化**
- 角色編輯面板從「30+ 欄位同時顯示」改為分步驟表單或可收合區塊

## 六、結語

天命共振的後台管理系統功能覆蓋面相當廣泛，從用戶管理到遊戲內容管理一應俱全，這是非常扎實的基礎。目前最需要的不是新增功能，而是**整合與打磨**——讓 15 個頁面看起來、用起來像「同一個系統」，而不是 15 個獨立的工具。

統一佈局和導航是投入產出比最高的一步，做完之後整個後台的專業感會有質的飛躍。
