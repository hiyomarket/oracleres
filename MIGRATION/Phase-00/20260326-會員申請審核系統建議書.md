## 🎯 對天命共振的建議

### 推薦方案：**Discord OAuth + 邀請碼制 + hCaptcha**

**理由：**
1. **Discord OAuth**：天命共振目標用戶是 Discord 社群成員，有第三方信任
2. **邀請碼制**：從資料表可見已有 `invite_codes` 機制，可控制會員品質
3. **hCaptcha**：防止機器人註冊

## 🎮 對墟界的建議

### 推薦方案：**Discord OAuth + Email 驗證**

**理由：**
1. **Discord OAuth**：遊戲玩家通常是 Discord 用戶
2. **Email 驗證**：基本要求，確保是真實 email
3. **簡化流程**：放置型遊戲不需要高強度審核

## 🏗️ 推薦會員系統架構

### 技術實現

| 層面 | 技術方案 |
|------|----------|
| OAuth | Passport.js + Discord/Google Strategy |
| 驗證 | JWT (Access Token 15分鐘 + Refresh Token 7天) |
| 防機器人 | hCaptcha |
| 邀請碼 | 自行開發或用現有 invite_codes 表 |

### 會員資料表設計（擴展現有）

```sql
CREATE TABLE `user_verifications` (
  `id` int NOT NULL AUTO_INCREMENT,
  `user_id` int NOT NULL,
  `verification_type` enum('email','discord','google','apple') NOT NULL,
  `verified_at` timestamp NULL DEFAULT NULL,
  `status` enum('pending','verified','rejected') NOT NULL DEFAULT 'pending',
  PRIMARY KEY (`id`)
);

CREATE TABLE `invite_codes` (
  `id` int NOT NULL AUTO_INCREMENT,
  `code` varchar(20) NOT NULL,
  `created_by` int NOT NULL,
  `used_by` int DEFAULT NULL,
  `used_at` timestamp NULL DEFAULT NULL,
  `expires_at` timestamp NULL DEFAULT NULL,
  PRIMARY KEY (`id`)
);
```
