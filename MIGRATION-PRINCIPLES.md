# 天命共振系統 ── GD-027/028 基礎改動最高原則

> **本文件是所有改動的最高指導原則。每完成一個步驟，必須回來比對本文件，確認沒有遺漏或走偏。**

---

## 核心信條

1. **「從人物、寵物、魔物都要有相對應的屬性」** ── 不是只改一個表，是所有圖鑑（人物/寵物/魔物/裝備/技能）的欄位都要一次到位。
2. **「命格是從用戶自身的八字五行比例按比例帶入遊戲」** ── 不是選擇命格，是直接讀取 `userProfiles.natalWood/Fire/Earth/Metal/Water` 的真實比例。
3. **「寵物＝魔物捕捉而來，五行分配固定」** ── 寵物的五行比例繼承自魔物圖鑑，不可更動。抓到都是 Lv.1，BP 隨機。
4. **「所有數據都要有後台可調」** ── 不要有系統硬編碼的數值，全部可以在後台管理介面修改。到時變數都沒得改，又要一次次的請系統改這個小地方。

## 步驟 2：屬性公式引擎（骨架）

建立 `server/services/statEngine.ts`，實裝 GD-028 的所有公式：

```
HP   = Lv×12 + 80 + (木÷100)×30 + 潛能分配
MP   = Lv×8  + 40 + (水÷100)×20 + 潛能分配
ATK  = Lv×8  + 15 + (火÷100)×30 + 潛能分配
DEF  = Lv×8  + 15 + (土÷100)×30 + 潛能分配
SPD  = Lv×6  + 10 + (金÷100)×20 + 潛能分配
MATK = Lv×8  + 15 + (火÷100)×25 + (水÷100)×10 + 潛能分配
MDEF = Lv×6  + 10 + (土÷100)×20 + (水÷100)×10
暴擊率   = 5 + (金÷100)×2
暴擊傷害 = 150 + (火÷100)×10
```

- 角色面板完整呈現所有屬性
- 人物潛能點數分配 UI + Router（每級+5點，分配至 HP/MP/ATK/DEF/SPD/MATK）
- 每點效果：HP+20, MP+10, ATK+3, DEF+3, SPD+2, MATK+3

## 步驟 3：命格系統（特色 ── 以身入遊戲）

**核心：直接讀取用戶 `userProfiles` 的八字五行比例，按比例帶入遊戲角色。**

- 讀取 `natalWood/Fire/Earth/Metal/Water`（例如：木30/火25/土15/金20/水10）
- 按比例分配到角色的五行初始值
- 自動判定命格主屬性（最高五行 → 對應命格）
  - 木最高 → 青龍命 → HP+10%, HEAL+5%
  - 火最高 → 朱雀命 → ATK+10%, MATK+10%
  - 金最高 → 白虎命 → SPD+10%, 暴擊率+5%
  - 水最高 → 玄武命 → MP+10%, 元素抗性+5%
  - 土最高 → 麒麟命 → DEF+10%, MDEF+10%
- 命格加成融入 statEngine 的公式計算
- 命格屬性技能經驗 +50%

## 步驟 5：傷害公式重構 + 種族相剋更新

- 物理傷害：`max(1, ATK×1.5 - DEF×0.5)` × 暴擊 × 五行剋制 × 種族相剋
- 魔法傷害：`max(1, MATK×1.5 - MDEF×0.5)` × 精神係數 × 五行剋制
- 治療公式：`技能等級 × 60 × 回復力%`
- 更新種族相剋表為兩條鏈：
  - 鏈1：人形 → 野獸 → 植物 → 不死 → 人形
  - 鏈2：龍 → 飛行 → 昆蟲 → 特殊 → 金屬 → 龍
- 所有公式中的常數都從後台可調的設定表讀取

## 步驟 6：寵物系統整合（寵物＝魔物捕捉）

- 捕捉時：從魔物圖鑑繼承五行分配（固定不變）、種族、基礎 BP
- BP 隨機分配（在圖鑑基礎值上下浮動）
- 抓到的都是 Lv.1
- 捕捉機率從魔物圖鑑的 `baseCaptureRate` + `isCapturable` 讀取
- 寵物經驗跟隨主人：出戰 100%、待命 30%
- 命格相合：主人命格五行＝寵物主五行 → 經驗+20%
- 寵物境界上限：初界 Lv.30、中界 Lv.60、高界 Lv.99
- 寵物 BP 每級 +3 點
